檔案與儲存空間

「檔案」頁是你所有上傳內容的統一管理入口:側欄「繪圖」下方點選「檔案」進入。對話裡發的附件、知識庫裡的文件,都在這一頁列出——不用再回到各個對話或知識庫裡逐個找。

頁面構成​

頂部是儲存用量條:顯示已用空間與總配額(由管理員按使用者組配置),用量超過 90% 時進度條變紅提醒。若你所在的組未設配額,則顯示「已用 xx · 無限制」。

下方是檔案列表,每行包括:

列說明
檔名帶型別圖示,懸停顯示完整名稱
型別副檔名(PDF / DOCX / XLSX / MD / PNG…)
來源「對話」徽標,或知識庫名稱徽標
大小檔案位元組數
上傳時間本地化時間
操作檢視(線上預覽)/ 刪除

支援按檔名搜尋(不區分大小寫)、按來源篩選(對話 / 知識庫)、按時間或大小排序,列表分頁載入。

線上預覽​

點選「檢視」直接預覽,無需下載:圖片直接顯示,PDF 內嵌閱讀器,文本/程式碼高亮展示(超過 2MB 的文本建議下載後檢視),其他格式提供下載。

儲存配額規則​

  • 只有文件類檔案計入配額:PDF、Word、表格、文本等。圖片不計入——聊天裡發再多圖也不佔空間。
  • 同一份上傳在對話與檢索索引間共享同一份位元組,只計一次。
  • 空間滿時繼續上傳文件會被拒絕,提示「儲存空間已滿,請先到檔案頁清理」。清理後即可繼續上傳。
  • 配額大小由管理員在「管理後臺 > 使用者組」按組設定(0 表示不限制)。

刪除檔案​

點選刪除並確認後,檔案會在所有位置被徹底移除:磁碟上的檔案本體、資料庫記錄、檢索向量一併清除,不可恢復。

刪除後相關對話不會報錯,但有兩點值得注意:

  • 歷史訊息裡的附件標籤仍然顯示,但點選預覽會提示檔案不存在;圖片附件的縮圖會變成「檔案已刪除」佔位。
  • 模型會"失憶":後續在該對話繼續提問時,模型不再能讀取這份檔案的內容(圖片不再提供給視覺模型、文件不再出現在檢索結果裡),只能依據之前輪次已生成的回答文字。知識庫文件同理——刪除後掛載該庫的對話檢索不到這份文件,其餘文件不受影響。
