圖片生成

Aivory 的圖片生成不是一個孤立的「畫圖頁面」,而是與聊天深度融合:在同一個對話方塊裡選中圖片模型即可繪圖,畫完切回聊天模型還能繼續用 Python 沙箱處理這張圖。本頁覆蓋繪圖入口、風格與提示詞最佳化、參考圖編輯、限額與積分計費、個人畫廊,以及管理員側的每使用者相簿。

前置:圖片模型從哪來​

圖片模型與對話模型共用同一套渠道 / 模型體系:管理員在某個渠道下建立 kind='image' 的模型條目即可(配置方法見渠道與模型)。兩類渠道的行為不同:

渠道型別生成方式編輯 / 改圖方式
Gemini對話式生成(generateContent 影像輸出)把「改成紫色天空」作為下一輪追加進同一影像會話,基於上一張圖修改,主體一致性強
OpenAI無狀態呼叫 /v1/images/generations走 /v1/images/edits,傳原圖與指令

使用者不需要感知這些差異,平臺統一吸收;多輪編輯的會話狀態持久化在服務端,重新整理頁面、隔天再改都能續上。

兩種使用方式​

方式一:繪圖模式(推薦)​

進入方式任選其一:

  • 側欄「繪圖」入口(專案區下方):開啟帶繪圖預設的首頁,自動選中第一個可用圖片模型;
  • 模型選擇器:任意對話中開啟模型選擇器,底部的「圖片生成」分組列出全部已啟用圖片模型(帶圖示與名稱),選中即進入繪圖模式。

繪圖模式下,傳送一條訊息就等於生成一張圖。生成過程有專屬的進度 UI:先顯示「正在最佳化提示詞…」,再顯示「正在繪製圖片…」,配畫布佔位動畫,完成後圖片直接出現在對話裡。

生成後想繼續加工(裁剪、加水印、拼圖、資料視覺化疊加),在模型選擇器切回聊天模型即可:同一個會話,剛生成的圖片對 Python 沙箱可見。

方式二:聊天中讓模型畫圖​

聊天模型可以通過 image_generate 工具畫圖:直接在普通對話裡說「幫我畫一張…」,模型判斷需要畫圖時自動呼叫工具。此路徑使用哪一個圖片模型,由你在「設定 → 模型 → 預設圖片模型」裡預選(沒有可用圖片模型時該處會提示聯絡管理員配置)。

單輪對話中該工具的呼叫上限為 8 次(深度研究模式下為 4 次)。

兩條路徑共用同一額度池

繪圖模式和聊天工具呼叫最終走的是同一個生成管線,對同一個圖片模型共用同一份每日限額與積分計費,不存在「換個入口繞過限額」。

提示詞最佳化與風格​

提示詞最佳化​

管理員可以在管理後臺「設定」頁配置「圖片提示詞模型」(一個文本模型):配置後,你的原始描述會先由它擴寫成一句更具體的繪圖提示詞再送去生成;未配置或調用出錯時跳過潤色、直接使用原文,保證總能出圖。

風格預設​

繪圖模式下輸入框會出現「風格」選擇器(縮圖網格):

  • 風格由管理員在管理後臺「圖片風格」頁維護,每個風格包含名稱、示例圖、排序與一段對使用者隱藏的風格提示詞;
  • 你只能看到風格名稱和示例縮圖,選中後服務端把隱藏提示詞拼進最終繪圖指令,提示詞本身絕不下發到瀏覽器;
  • 選「無」則不套任何風格;管理員未配置風格時顯示「暫無風格」。

重新生成某張圖時會沿用原來選擇的風格,不需要重選。

參考圖:圖生圖與編輯​

在繪圖模式下上傳圖片附件,它就會作為輸入圖參與生成:

  • 上傳自己的照片 + 文字指令 = 圖生圖 / 風格遷移;
  • 對上一輪生成的圖繼續發修改指令 = 多輪編輯(Gemini 類模型在多次「再改一點」之間保持人物與場景一致);
  • 圖片附件有大小上限(管理員可配,預設 5 MB),超限會在選擇檔案時即被提示攔截。

尺寸與數量​

生成介面支援數量(n)與尺寸(size)引數:

  • 聊天工具路徑:直接把要求寫進指令,例如「畫 2 張橫版 16:9 的海報草圖」,模型會把它翻譯成工具引數;尺寸引數是否生效取決於上游圖片模型自身的支援情況。
  • 繪圖模式:構圖要求(橫版 / 豎版 / 比例 / 張數)同樣寫在提示詞裡,提示詞最佳化模型會把它保留進最終指令。

限額與積分計費​

圖片單價高,平臺對它的管控比訊息更細,規則按以下順序生效:

  1. 每日張數上限:管理員在「設定」頁配置全站的「每日圖片上限」,與每日訊息上限相互獨立。
  2. 按組按模型的免費額度:管理員可為每個圖片模型對每個使用者組配置週期免費額度。額度內生成免費。
  3. 積分回退:免費額度用完後自動改為扣積分,按「單張價格 × 每美元積分數」逐張折算,與聊天共享同一積分餘額;積分也不足時拒絕生成並提示。
  4. 豁免與封禁:管理員賬號不受額度限制;某圖片模型若對你所在的使用者組沒有開放,模型選擇器裡不會出現它。

配套的可見性設計:

  • 當你所在組的免費額度用完、生成將要扣積分時,模型選擇器「圖片生成」分組裡該模型名稱右側會顯示「N 積分」徽標(單張成本),心裡有數再畫;
  • 計費時點是「出圖才算」:生成失敗、被攔截不計次不扣分;
  • 每張圖的消耗計入用量記錄,管理員可在用量報表中按「圖片」用途篩選。

額度與積分體系的完整說明見使用者與配額,常見疑問見常見問題中「每個使用者每天能發多少條訊息、生成多少張圖」條目。

生成超時

管理員可為每個圖片模型單獨設定「生成超時(秒)」,單次請求超過該時長即中斷(0 為不限)。遇到慢模型頻繁超時,請管理員調大該值。

我的畫廊​

從側欄「繪圖」進入繪圖首頁後,向下滾動即是「我的畫廊」:

特性說明
內容你的全部生成圖片,繪圖模式與聊天工具呼叫產出的都在
佈局瀑布流版面,滾動進入視口時逐張浮現
載入懶載入:滾動到畫廊區域才發起第一頁請求;之後自動無限滾動分頁,每張圖有獨立骨架佔位
跳轉點任意縮圖跳回該圖所在的對話,可以繼續編輯或追問
空態顯示「還沒有作品,在上方畫一張吧」

畫廊與對話裡的生成圖都啟用了瀏覽器端快取(HTTP 快取 + Service Worker 的圖片 cache-first),重複瀏覽不再回源拉圖;圖片列表本身始終從服務端取最新。

常見問題排查​

模型選擇器裡沒有「圖片生成」分組

管理員尚未建立任何 kind='image' 的模型,或已建立但未啟用、未對你所在的使用者組開放。圖片模型對某個組不可用時,該組使用者的選擇器里根本不會出現它。請管理員到「渠道 / 模型」頁檢查,見渠道與模型。

「設定 → 模型 → 預設圖片模型」顯示暫無可用

同上,本質是沒有對你開放的圖片模型。此時聊天模型的畫圖工具也無從生效。

生成長時間停在「正在繪製圖片…」後失敗

單次生成受該模型的「生成超時(秒)」限制,超時即中斷;上游模型排隊或網路慢時容易觸發。稍後重試,或請管理員調大該模型的超時值。生成失敗不計入限額、不扣積分,可放心重試。

提示額度已用完或積分不足

先核對三層限制:全站每日圖片上限、你所在組對該模型的免費額度、積分餘額。管理員可在「設定」頁與使用者組配額裡調整,使用者可通過兌換碼補充積分。詳見使用者與配額與常見問題。

畫廊裡圖片一直空著不載入

畫廊是懶載入的,需要滾動到畫廊區域才發起請求;單張圖有獨立的骨架佔位,載入完才淡入。若長期空白,檢查網路與登入狀態後重新整理頁面。

管理員側:每使用者相簿​

管理員在管理後臺「使用者」列表中,通過使用者行的更多選單進入該使用者的資料庫頁,其中「相簿」區以網格展示該使用者的全部生成圖片(繪圖模式與聊天工具呼叫兩條路徑的產出都覆蓋):

  • 點任意縮圖直達該圖所在對話的管理員檢視頁,可以看到完整的生成上下文(提示詞、參考圖、多輪編輯過程);
  • 列表支援「載入更多」分頁;
  • 配合用量報表的「圖片」用途篩選,可以核對每使用者的生成量與費用。
