分享與資料

本頁介紹 Aivory 中與「你的資料」相關的全部能力:把對話生成公開連結分享給他人、匯出個人資料、從其他平臺或本站匯入歷史對話、隱私設定頁,以及跨對話記憶的開啟與管理。

對話分享​

分享連結是什麼​

分享連結是一份只讀、公開、無需登入的對話快照:

  • 分享的內容是當前活躍路徑上的訊息(即你此刻在聊天窗口裡看到的那條線),不包含其他分支。
  • 快照在建立的一刻定格:之後的新訊息不會出現在分享中。想讓新內容可見,再次執行一次分享即可:新快照替換舊快照,同時生成新的連結,舊連結隨之失效。
  • 快照只保留使用者與助手訊息的正文、引用與附件資訊,不攜帶你的賬號身份資料。
注意

分享連結是公開的:任何拿到連結的人都可以檢視,無需登入、無需審批。不要分享包含敏感資訊的對話。

建立分享連結​

有兩個入口:

入口操作路徑行為
對話頁頂欄三點選單 → 「分享」開啟「分享對話」彈窗,點「建立分享連結」後顯示連結、複製按鈕與「公開,無需登入即可檢視」提示,並提供「取消分享」按鈕
側邊欄歷史列表對話行的三點選單 → 「分享」一步完成:建立(或重新整理)分享並直接把連結複製到剪貼簿,彈出「分享連結已複製」提示

移動端在對話頁頂欄的操作面板(action sheet)裡同樣有「分享」項。

訪客看到的分享頁​

分享頁地址形如 https://你的域名/share/<分享ID>,開啟後:

  • 使用與聊天頁完全一致的 Markdown、程式碼塊、引用來源渲染,但隱藏所有操作按鈕(傳送、重試、編輯等)。
  • 頁面頂部顯示「分享的對話」眉標與對話標題,訊息按「使用者 / 助手」角色排列。
  • 頁尾顯示「由 Aivory 分享」水印;未登入訪客會看到「體驗 Aivory」入口。
  • 已登入使用者會看到「克隆到我的對話」按鈕:一鍵把這份快照複製成自己賬號下的一個新對話,之後可以在其基礎上繼續提問。

附件與生成圖片(帶資產分享)​

分享快照會攜帶訊息中的資產,匿名訪客也能正常檢視:

資產型別分享頁中的呈現
使用者上傳的圖片附件內聯顯示為圖片
使用者上傳的其他檔案顯示為檔案卡片(檔名)
AI 生成的圖片(如 image_generate 產物)內聯顯示為圖片
AI 生成的其他產物顯示為下載連結

訪客對這些檔案的訪問走分享專屬的公開路由,並且只能訪問該快照裡出現過的檔案,無法借分享連結列舉你的其他上傳內容。

提示

如果你的部署是從舊版本升級上來的:舊分享的快照裡沒有上傳附件欄位,需要重新分享一次才能讓上傳檔案出現在分享頁;AI 生成圖片則無需重新分享。

撤銷分享​

在「分享對話」彈窗裡點「取消分享」:

  • 撤銷立即生效,原連結馬上變為 404(分享頁顯示「該分享不可用」)。
  • 撤銷後可以隨時再次建立分享。

安全性​

  • 分享連結中的 ID 是高熵隨機令牌(192 位),無法被暴力猜測,拿不到連結就看不到內容。
  • 分享頁不洩露分享者的賬號身份資訊。
  • 快照中的檔案訪問有獨立限速,防止濫用。

匯出個人資料​

路徑:頭像選單 → 設定 → 隱私與資料 → 「匯出全部資料」。

點選「匯出」後,瀏覽器會下載一個 JSON 檔案,檔名形如 aivory-export-2026-07-11.json,包含:

欄位內容
conversations你的全部會話,每個會話含標題、後設資料與完整訊息列表
memories你的全部記憶條目
exported_at匯出時間(ISO 8601)

匯出在你的瀏覽器本地組裝並直接下載,內容嚴格限定為當前登入賬號自己的資料。

資訊

這份匯出檔案可以直接被本頁的「匯入對話」重新識別並匯入(見下節),因此也可以當作跨部署搬運個人對話的媒介。

匯入對話​

路徑:設定 → 隱私與資料 → 「匯入對話」,選擇一個 .json 檔案。

支援的格式(自動識別)​

同一個入口接受兩種格式,解析器會自動判別,無需手動選擇:

格式說明額外能力
其他平臺的聊天匯出訊息以帶父子關係的對映結構儲存的通用 JSON 匯出編輯/重試產生的分支樹會完整遷移為 Aivory 的對話樹
本站「匯出全部資料」檔案上一節匯出的 { conversations: [...], memories, exported_at } 結構額外攜帶每個會話的 model_id,匯入後可直接用原模型續聊

匯入範圍與清洗​

匯入只保留聊天曆史與標題,其餘內容在瀏覽器端解析時全部剔除:

  • 上傳的檔案與用量記錄不匯入。
  • 助手正文裡的思考/狀態摺疊塊、內部推理標記、內嵌 base64 圖片全部剝離。
  • 本站格式只取文本塊拼接正文,思考、工具呼叫、引用、圖片塊按同一口徑丟棄;system 訊息跳過。

分支樹遷移​

原始檔裡的編輯分支(包括對首個提問的編輯產生的多個根)會原樣遷移:匯入完成後,分支點上會出現 < n/m > 切換控制元件,活躍分支也會對齊原始檔當時停留的位置;活躍葉子失效時自動回退到最後一條訊息。

限額與結果​

限額項上限
每次匯入的會話數1000
每個會話的訊息數10000
單條訊息正文200 KB
  • 匯入不呼叫任何模型、不消耗積分與額度,純資料寫入。
  • 按會話粒度容錯:某個會話解析失敗會被跳過,不影響其餘會話;完成後提示「已匯入 N 個對話」,若有失敗會附帶失敗數量。
  • 原始檔裡被中止的空助手回覆以「已停止」狀態落庫,不會在介面上誤報錯誤。

隱私設定頁​

路徑:設定 → 隱私與資料。頁面分為兩個區塊:

區塊內容
資料儲存說明性文字:對話儲存在你所連線的伺服器上(自部署場景即你自己的伺服器);如需刪除資料可聯絡管理員
匯出與清除「匯入對話」「匯出全部資料」「清除全部會話」三行操作
危險

「清除全部會話」會永久刪除你的所有會話,點選後需在確認彈窗中再次確認,刪除後不可恢復。如果只想刪除個別對話,請在側邊欄對該對話單獨操作。

記憶的開關與管理不在本頁,而在「設定 → 個性化」,見下節。

記憶​

記憶是什麼​

Aivory 可以跨對話記住關於你的長期事實(偏好、身份、約束等):

  • 每輪對話結束後,後臺由任務模型非同步提取候選記憶,不佔用你等待回答的時間。
  • 記憶是狀態感知的:新事實寫入時會自動裁決舊記憶是否過期(例如「搬到東京了」會讓「住在北京」變為過期),過期記憶只作歷史參考,不再當作當前事實。

記憶條目的狀態含義:

狀態介面標籤含義
ACTIVE有效當前事實,回答時直接採用
STALE過期僅作歷史,不作當前依據
UNKNOWN_CURRENT未知當前是否仍成立不確定
HISTORICAL_ONLY僅歷史只用於「我以前……」類問題
QUERY_DEPENDENT結合問題判斷是否適用取決於當前提問

開啟與關閉​

路徑:設定 → 個性化 → 「記憶」區塊。

  • 「啟用記憶」開關控制記憶的使用與儲存:關閉後,所有對話既不注入已有記憶,也不再提取新記憶。
  • 該偏好儲存在服務端,跨裝置生效。
  • 管理員可以在平臺層面全域性關閉記憶功能;全域性關閉時,整個「記憶」區塊不會出現在你的設定裡。

管理記憶​

「啟用記憶」開啟後,同一區塊內顯示記憶管理器:

操作說明
瀏覽列出全部記憶,每條帶狀態徽章與可選的屬性鍵
篩選按「全部 / 有效 / 過期」過濾
新增記憶手動新增:填寫記憶內容,可選屬性鍵與屬性值
編輯修改記憶內容或狀態(標為過期 / 標為有效)
遺忘刪除該條記憶,需確認

聊天中的記憶​

  • 你可以在對話裡直接說「記住……」,模型會通過 save_memory 工具顯式寫入一條記憶,聊天介面會提示記憶已更新。
  • 回答時,有效記憶會作為系統提示的一部分注入,讓模型「認識你」;這不需要任何額外的模型呼叫,不影響回答速度。

隱私邊界​

  • 工作空間中的共享對話完全不使用記憶:既不注入你的個人記憶,也不從共享對話中提取,防止你的個人事實洩露給空間成員,詳見 團隊工作空間。
  • 記憶包含在「匯出全部資料」的 memories 欄位中,可隨時匯出。
  • 每條記憶都可以單獨刪除;刪除賬號時記憶級聯清除。
