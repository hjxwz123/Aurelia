深度研究

深度研究(Deep Research)是一條獨立的多輪研究流水線:模型先制定研究計劃,再執行多輪網路搜尋與頁面精讀,對斷言做交叉驗證,最終生成一份帶引用來源的完整報告。與普通問答相比,它的工具預算大幅提升,適合需要多來源佐證的綜述型問題。

開啟方式​

  1. 在輸入框工具欄點望遠鏡圖示的「研究」按鈕,按鈕點亮為鼠尾草綠即已開啟。
  2. 正常輸入問題併發送,該條訊息即走深度研究流程。
  3. 開關保持開啟狀態,後續訊息會繼續走研究模式,直到你再點一次關閉。

「研究」按鈕的顯示需要同時滿足三個條件:

條件由誰控制
當前模型開放了深度研究管理員在模型編輯頁的深度研究暴露開關,見 渠道與模型
你所在使用者組具備研究許可權管理員在使用者組的功能開關中配置
當前不在繪圖模式選中圖片模型時研究按鈕自動隱藏
資訊

移動端「研究」入口收在輸入框的「+」選單裡;研究開啟時「+」按鈕顯示高亮圓點。

與普通模式的區別​

普通問答裡模型也能聯網搜尋,但預算有限、不做系統性驗證。深度研究把單輪工具預算整體抬高:

工具普通模式上限深度研究模式
網路搜尋(web_search)1640
頁面抓取(web_fetch)1225
圖片下載(fetch_image)1612
圖片生成(image_generate)84
Python 執行(python_execute)168
全工具合計48150

除了預算,流程也不同:普通模式是「模型想搜就搜」;研究模式是「先拆解計劃,再按計劃覆蓋每個子問題,最後統一驗證與撰寫」。

流程解釋​

一次深度研究分為幾個階段,全程由引擎自動推進:

  1. 規劃:把你的問題拆解為 3 到 6 個互補的子問題,識別研究型別(概念解釋、對比、趨勢、技術、市場、決策),並確定報告結構。計劃中會刻意包含至少一條反面或批評視角的查詢,避免資訊繭房;涉及時效的查詢自動帶上年份,技術類話題至少包含一條英文查詢。
  2. 廣度搜索與精讀:圍繞每個子問題並行執行多輪網路搜尋,並抓取頁面全文精讀。已訪問過的頁面自動去重,閱讀順序優先選擇可信度更高、域名更多樣的來源。
  3. 覆蓋審計:檢查每個子問題是否有足夠的獨立來源支撐,標記證據薄弱或未覆蓋的子問題,並補充新一輪搜尋。
  4. 交叉驗證:把收集到的證據分為三類,直接決定報告中的措辭:
    • 已確認:兩個以上獨立來源一致,報告中直接陳述;
    • 有爭議:來源之間說法衝突,報告分別呈現各方觀點,不強行合併;
    • 未驗證:只有單一來源,報告中註明「據某來源」。
  5. 撰寫報告:主模型流式生成最終報告,邊寫邊渲染。

來源可信度分級​

每個來源按域名啟發式評為 A 到 D 四級,顯示在進度面板的來源卡片上:

等級含義
A官方、學術、標準組織
B權威媒體、研究機構
C部落格、社群、未知域名
D匿名論壇、無署名內容

A / B 級徽章以鼠尾草綠高亮,引擎在閱讀排序上優先消化高可信來源。

進度介面​

傳送後,回覆上方出現帶望遠鏡圖示的研究面板,研究期間自動展開、即時更新:

  • 標題行:研究主題、進度計數 N/M(已完成子問題數 / 總數)。
  • 研究計劃:子問題清單,每項前有狀態標記:灰色圓圈為待研究,鼠尾草綠脈衝點為進行中,綠色對勾為已完成。
  • 來源(N):雙列來源卡片,顯示標題、域名與 A 到 D 可信度徽章;抓取失敗的來源以警示三角標出。點選卡片在新標籤頁開啟原文。

報告正文開始輸出後,面板自動摺疊為一行摘要,隨時可點開回看完整計劃與來源列表。

產出:帶引用的報告​

最終報告作為訊息正文流式呈現,結構隨研究型別變化(對比類含總覽表與場景建議表,趨勢類含指標表與短中長期展望),共同遵守:

  • 開頭是元資訊行(研究日期、範圍說明)與概述;
  • 關鍵發現逐條編號;
  • 已確認的結論直接陳述,有爭議的呈現各方觀點,單源資訊標註出處;
  • 資料標註時間;
  • 明確寫出研究侷限;
  • 文末附參考來源列表。

報告是一條普通的 AI 回覆:可以複製、重新生成(重跑一輪研究,開新分支)、繼續追問、劃詞開子對話,也會被對話分享連結包含。研究面板與來源列表隨訊息持久儲存,重新整理頁面後仍可回看。

中斷與重跑​

  • 研究進行中可隨時點「停止生成」:已產出的內容保留;按平臺「發出即算」的計費規則,已消耗的部分正常計入用量與積分。
  • 對報告點「重新生成」會完整重跑一輪研究(重新搜尋與驗證),以平行分支儲存,用 < N/M > 控制元件與舊報告對照。

適用場景與耗時​

適合:

  • 需要多來源交叉驗證的事實綜述("X 領域近兩年的主要進展");
  • 對比選型("A 與 B 方案各自的優劣與適用場景");
  • 趨勢與市場分析;
  • 希望拿到可核查引用清單的調研任務。

不適合:

  • 簡單事實問答或閒聊(普通模式更快,一兩次搜尋即可);
  • 純程式碼、純計算任務(交給 Python 沙箱);
  • 只依賴你自己文件的問題(掛載 知識庫 即可)。
耗時與成本提示

深度研究要執行幾十次搜尋與抓取,再做驗證與撰寫,耗時明顯長於普通問答,通常以分鐘計。期間可以隨時點「停止生成」中斷。研究過程呼叫的任務模型與主模型均正常計費,消耗的額度與積分高於普通對話,請按需開啟。

管理員側配置​

深度研究依賴兩項後臺配置,普通使用者如果看不到「研究」按鈕或搜尋始終失敗,請聯絡管理員檢查:

  1. 搜尋後端:需要在後臺配置 SearXNG(自部署)或 Serper 相容的搜尋服務;未配置時研究無法獲取證據,報告會降級為無引用的普通回答。
  2. 模型與使用者組開關:按模型開放研究入口、按使用者組授予研究許可權,配置方法見 渠道與模型。

常見問題排查​

現象可能原因
輸入框沒有「研究」按鈕當前模型未開放研究,或你所在使用者組無研究許可權,或當前選中的是圖片模型
報告沒有引用來源管理員未配置搜尋後端,研究降級為普通回答
進度面板長時間停在同一子問題目標站點抓取緩慢或失敗,失敗來源會以警示三角標出,引擎會自動換來源繼續
報告明顯比預期短中途點過「停止生成」,或問題本身證據稀少(報告的「侷限」段會說明)
