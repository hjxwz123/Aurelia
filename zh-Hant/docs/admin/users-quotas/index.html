使用者、配額與積分

本頁覆蓋「使用者」大類下的三個標籤頁:使用者(/admin/users)、使用者組(/admin/user-groups)、兌換碼(/admin/redeem-codes),以及與之聯動的按組模型配額、積分系統和每日限額。

使用者管理​

列表與搜尋​

  • 搜尋:按名稱或郵箱模糊搜尋,輸入停止 400 ms 後自動提交(服務端過濾,大使用者量下依然完整)。
  • 分頁:服務端分頁,每頁 50 條。
  • 線上狀態:每行左側的圓點,綠色表示線上。判定標準:最近 5 分鐘內發起過帶認證的請求(活躍時間最多每分鐘重新整理一次)。離線使用者顯示「上次線上」時間,從未登入的顯示「從未登入」。
  • 行內徽章:角色(普通使用者/管理員)、非預設使用者組名、非 active 狀態(如封禁)、以及標記自己賬號的「你」。
  • 列表支援拖拽調整顯示順序。

新建賬號​

點「新建使用者」,填寫:

欄位要求
郵箱必填,需為有效郵箱格式
名稱選填
密碼至少 8 個字元
角色普通使用者 / 管理員

管理員直接建號免郵箱驗證,適合內部部署或為同事開號。

編輯賬號​

每行的鉛筆圖示開啟編輯彈窗:

欄位說明
角色普通使用者 / 管理員;不能修改自己的角色
使用者組決定該使用者可用的模型與額度
會員到期日期選擇,留空表示永久;到期後回落到之前的使用者組
兩步驗證僅當該使用者已啟用 2FA 時出現「重置兩步驗證」按鈕,用於使用者丟失驗證器裝置的場景
重置密碼輸入新密碼(至少 8 字元)即重置;留空保持原密碼
永久積分直接改寫該使用者的永久積分餘額

資訊圖示(檢視資訊)開啟只讀面板:郵箱、角色、狀態、使用者組與到期時間、永久積分、所在組的積分額度、兩步驗證狀態、最近活躍、建立時間。

封禁、解封與刪除​

  • 封禁是即時全域性生效的:舊 token 立即失效、進行中的生成被斷流、refresh token 作廢,機制詳見後臺總覽。解封按鈕在同一位置。
  • 刪除使用者有確認彈窗,不可撤銷。刪除是非同步的:確認後賬號立即被鎖定登出(與封禁同機制),列表中顯示「刪除中」徽標,重活(對話、訊息、檔案、向量、磁碟清理)由後臺任務分批完成,完成後該行自動消失——大賬號的刪除不再卡住頁面。刪除中的賬號不能被解封「復活」;若程序重啟,未完成的刪除會自動續跑。
  • 自我保護:不能封禁、刪除自己。

只讀鑽取​

每行「更多」選單提供兩條只讀鑽取路徑,用於濫用排查與支援工單:

  • 對話記錄:/admin/users/:id/conversations,列出該使用者全部會話(含已歸檔),點進單條會話可看完整訊息時間線,渲染與使用者端完全一致(含推理塊、工具卡片、引用、圖片),但不提供任何編輯/重生成入口。
  • 專案與知識庫:/admin/users/:id/library,該使用者的專案、知識庫文件與生成相簿(相簿分頁載入)。

使用者組​

使用者組是會員分層的載體:名稱、價格與功能清單展示在使用者的訂閱頁;模型許可權與用量額度則在模型編輯頁按組配置(見下節)。

組欄位​

名稱如 Free / Pro / VIP
簡短介紹展示在訂閱頁的一句話

最大專案數 / 最大知識庫數 / 最多工作空間數各自 0 = 不限

  • 刪除其他組時,組內成員移至預設組,不可撤銷。
  • 列表支援拖拽排序,影響訂閱頁的展示順序。

組列表下方的全域性項​

同頁還承載四個全域性設定:

設定說明
每美元積分數(全域性)credits_per_usd:1 美元等於多少積分,是積分系統的總開關,0 = 停用積分
使用者組購買連結(全域性)訂閱頁付費組卡片「購買」按鈕的跳轉地址;留空顯示「聯絡管理員」
積分補充連結(全域性)使用者購買永久積分的跳轉地址
超額提示使用者達到某模型用量上限時顯示的文案

按組模型配額​

在模型編輯頁(「模型 > 模型 > 設定」)的「許可權」分割槽,對 chat 與 image 模型逐組配置:

  • 授予開關:開啟表示該組可用此模型。一旦有任何組被授予,未授予的組就看不到/用不了該模型;全部關閉則模型對所有人無限開放。
  • 週期:每 N 天或 N 小時為一個固定視窗。
  • 上限型別:二選一。
    • 次數:視窗內最多呼叫多少次;
    • 額度:視窗內按模型配價累計的費用上限。
  • 上限值:0 表示該週期內無限制。

超出免費額度後的行為:若積分系統已啟用(credits_per_usd 大於 0)且使用者有餘額,該模型對這個使用者轉為按真實費用扣積分繼續使用;積分也不足、或積分系統未啟用時,請求被攔截並顯示「超額提示」文案。管理員賬號與「無任何配額行的模型」不受此限。

免費 + 積分的組合玩法

典型配置:給 Free 組某模型「每 7 天 20 次」,給該組「每週期積分」發一些時效積分。使用者先消耗免費次數,用完後自動轉為扣積分,兩條額度互相獨立。

積分系統​

兩種積分​

型別來源生命週期
時效積分使用者組「每週期積分」按重新整理週期自動發放週期結束未用完即作廢
永久積分管理員在使用者編輯頁直接設定;或通過「積分補充連結」外部購買後由管理員充值永不過期

扣減順序:先扣時效積分,再扣永久積分。

換算與展示​

  • 扣費金額 = 該次呼叫按模型配價算出的費用(美元)× credits_per_usd。
  • 模型選擇器對走積分的對話模型顯示相對倍率徽標 ×N,N =(輸入價 + 輸出價)÷ 5,保留一位小數,讓使用者直觀比較各模型的貴賤。
  • 按張計費的圖片模型,當免費次數用盡轉為扣積分時,選擇器在模型名後顯示「N 積分」,N = 單張圖片價格 × credits_per_usd(例:一張 $0.2,credits_per_usd=100,顯示 20 積分)。
  • 使用者在訂閱頁可看到時效積分餘額(含過期時間)與永久積分餘額;每條 AI 回覆也記錄了本回合消耗。

計費時點:發出即算,失敗不算​

  • 正常完成:計入視窗配額次數,按全額扣積分。
  • 使用者中途停止:只要模型已產出內容,同樣計次,並按已產出 token 的真實費用扣積分。
  • 失敗或傳送前攔截(上游報錯、配額超限、內容稽核攔截):不計次、不扣分。

積分預檢開關​

「系統 > 全域性設定」中的「預估並攔截餘額不足的請求」(預設開啟):對走積分的回合,在真實請求裝配完成後、發給模型前,預估輸入 token(外加固定 2k 輸出預留)並換算成積分;若超出使用者可用餘額(時效 + 永久),直接以「餘額不足」拒絕,不產生任何上游消耗。積分系統停用時該邏輯自動跳過。

兌換碼​

兌換碼用於批次發放會員:每個碼授予一個使用者組與一段有效時長。使用者在訂閱頁輸入兌換碼完成兌換。

生成批次​

點「新建批次」:

欄位說明
對應使用者組兌換後用戶被加入的組
有效時長(天)兌換後會員持續天數;0 = 永久
單碼最大使用次數預設 1(一次性);調大可做多人共享的活動碼
兌換碼過期時間過期後未兌換即失效;留空 = 永不過期(注意這是碼本身的截止時間,不是會員時長)
批次名稱後臺內部標籤,便於篩選,使用者不可見
備註內部備註
生成數量1 到 1000,超出範圍被拒絕

生成後立即彈出本批次全部碼,可逐個複製;列表頁另有「複製全部未使用」與「匯出 CSV」(含碼、組、狀態、時長、已用次數、批次、備註、時間等列)。

管理與狀態​

  • 篩選:按狀態(全部 / 未使用 / 已兌換 / 已停用 / 已過期)與批次名過濾;列表每頁 20 條。
  • 狀態:未使用、部分使用(多次碼用了一部分)、已兌換(用滿)、已停用、已過期。
  • 停用/啟用:停用可逆,保留審計記錄,未兌換的碼立即失效。
  • 刪除:刪除單碼或整個批次,連同兌換記錄一起移除,不可撤銷;已經授予的會員不受影響,到期自然回落。
  • 兌換記錄:每個碼可檢視兌換審計(誰、何時、從哪個組切換)。

兌換行為與限流​

  • 兌換會把使用者立即切換到碼對應的使用者組(是覆蓋,不是疊加續期)。若碼的組與使用者當前組不同,服務端先返回確認要求,前端彈窗提示「將切換到 X 組」,使用者確認後才生效。
  • 到期後自動回落到兌換前的使用者組。
  • 防爆破限流:兌換介面按 IP 限流,預設 60 秒內最多 10 次,可用環境變數 AIVORY_API_RATE_LIMIT_REDEEM_CODE_MAX / AIVORY_API_RATE_LIMIT_REDEEM_CODE_WINDOW 調整,見進階環境變數。
兌換碼不直接充積分

當前版本的兌換碼授予的是使用者組 + 時長;積分通過使用者組的「每週期積分」隨組發放,永久積分由管理員在使用者編輯頁手工調整。想用兌換碼發積分,把積分額度配在碼對應的使用者組上即可。

每日訊息與圖片限額​

全域性的每人每日硬限額,與使用者組、積分互相獨立:

項環境變數(首次啟動種子)預設值後臺位置
每日訊息上限DAILY_MESSAGE_LIMIT200系統 > 全域性設定 > 每日訊息上限
每日圖片上限IMAGE_DAILY_LIMIT30系統 > 全域性設定 > 每日圖片上限

環境變數只在首次啟動時作為種子寫入設定表;此後執行期以後臺設定為準,改後臺即改行為,無需重啟。環境變數清單見核心環境變數。
