# Aurelia 完整功能文档

> 本文档基于对仓库**真实代码**的逐模块审读（后端 Go `server/`、前端 React `src/`、独立沙盒 `sandbox-service/`），按模块梳理现有全部功能点、页面、接口、数据流与关键逻辑，并标注每个功能的**真实完成度**（FULLY IMPLEMENTED / PARTIAL / UI-ONLY / STUB）。
>
> 配套的安全/质量审查见 [功能检查报告.md](功能检查报告.md)。

## 0. 架构总览

| 层 | 技术 | 位置 |
|---|---|---|
| 前端 | React 19 + TypeScript + Vite + Tailwind v4 + Zustand + react-router v7 + react-i18next | `src/` |
| 后端 | Go + 自研轻量 mux（`net/http`）+ SQLite/PostgreSQL（`pgcompat`）+ Redis（可选）+ Qdrant（可选） | `server/` |
| 代码沙盒 | 独立仓库：Python FastAPI sidecar + 每会话 Docker 容器 | `sandbox-service/`（独立 git） |
| 鉴权 | JWT（HS256）access(2h)+refresh(30d)，httpOnly Cookie + 内存 Bearer | `server/internal/auth` |
| LLM | 编排器 + Provider 抽象（Anthropic / OpenAI Chat&Responses / Google） | `server/internal/llm` |

- **数据库**：22 张表（`settings, users, user_groups, model_group_quotas, channels, models, skills, model_skills, knowledge_bases, projects, conversations, messages, conversation_shares, files, documents, chunks, memories, usage_logs, artifacts, refresh_tokens, oauth_providers, oauth_identities`）。SQLite/PG 双 schema（`schema.sql` / `schema_pg.sql`），启动时 `Migrate()`（含 `ALTER` 增量迁移）+ `Seed()` 自动执行。
- **后端自动检测后端依赖**：`postgres://` → PostgreSQL，否则 SQLite；`REDIS_URL` → Redis 否则内存缓存；`QDRANT_URL` → Qdrant 否则关系库暴力向量检索。
- **约 110 个 API 路由**，前端约 37 个页面。
- **无真实支付**：订阅/用户组价格为展示用，无任何 Stripe/支付集成。
- **成本对用户隐藏**：仅管理员可见花费；用户接口经 `redactCost` 清零 cost/currency。

## 1. 认证与账户安全（Auth）

**涉及**：`server/internal/auth/{auth.go,totp.go}`、`server/internal/api/{auth_handlers,twofa_handlers,oauth_handlers,middleware,user_handlers}.go`、`server/internal/oauth/oauth.go`、`server/internal/store/{users,oauth,misc}.go`、`src/store/auth.ts`、`src/pages/auth/*`、`src/components/auth/*`、`src/api/client.ts`

| 功能 | 接口 / 入口 | 完成度 |
|---|---|---|
| 注册（bcrypt、域名白名单、signup_open、邮箱验证可选） | `POST /api/auth/register` → `registerHandler` | ✅ FULLY |
| 登录（密码 + 状态校验） | `POST /api/auth/login` → `loginHandler` | ✅ FULLY |
| 2FA 登录第二步（TOTP，5min ticket） | `POST /api/auth/login/2fa` → `login2faHandler` | ✅ FULLY |
| TOTP 开启/确认/关闭 | `POST /api/me/2fa/{setup,enable,disable}` | ✅ FULLY |
| OAuth 登录（google/github/apple/oidc，PKCE，state） | `GET /api/auth/oauth/:id/{start,callback}` | ✅ FULLY（OAuth 也走 2FA） |
| JWT 签发/刷新（access 2h、refresh 30d、token_ver 实时封禁） | `POST /api/auth/refresh` → `refreshHandler` | ✅ FULLY |
| 登出（撤销 refresh jti、清 Cookie） | `POST /api/auth/logout` | ✅ FULLY |
| 找回密码（6 位码，10min，恒定 200 防枚举） | `POST /api/auth/{forgot,reset}-password` | ✅ FULLY |
| 邮箱验证码 | `POST /api/auth/{verify-email,send-code}` | ✅ FULLY |
| 改密（验证旧密码 + 踢登录） | `PATCH /api/me/password` | ✅ FULLY |
| 注销账户（密码确认） | `DELETE /api/me` → `deleteMeHandler` | ⚠️ PARTIAL（清理不全、未要 2FA） |
| 管理员重置某用户 2FA（丢设备找回） | `POST /api/admin/users/:id/2fa/disable` | ✅ FULLY |

**关键逻辑**
- TOTP 自实现 RFC 6238（SHA-1/6 位/30s，±1 步容错，`totp.go`），已用 RFC 4226 向量单测。
- `requireAuth`（`middleware.go:38`）每请求重载用户并校验 `token_ver` 与 `status==active`，支持实时封禁；封禁经 `user:<id>:kill` pub/sub 终止进行中的 SSE。
- Cookie：`HttpOnly + Secure + SameSite=Lax`；access 路径 `/`、refresh 路径 `/api/auth`。前端 access token 仅存内存，随 `Authorization: Bearer` 发送。
- 限流：`rateLimitedIP` 仅作用于 `/api/auth/*`（register 5、login 10、send-code 3、forgot 3…/min/IP）。

> 安全要点（详见检查报告 A 区）：OAuth 未验证邮箱也会联结既有账户（**严重**）、id_token 未验签、refresh 刷新不轮换/改密不撤销 refresh、无 CSRF、开发态默认 JWT 密钥/管理员密码。

- sidecar 以 **root** 运行并挂载 **`/var/run/docker.sock`**，对宿主机 Docker 有完全控制权。

- 每模型工具集不可细分：`registry.List(_)` 忽略 modelID，返回全部工具给所有模型；`python_execute` 无每轮次数上限。

> 这是全项目最高风险区，详见报告 F 区（CRITICAL：docker.sock+root；默认无鉴权 RCE）。
