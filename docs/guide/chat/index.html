Conversation and Model

This page introduces every element and operation of the Aivory chat interface to end users one by one: new conversation, model picker, all the capabilities of input boxes, stream output and thinking processes, tool call lines, message operations, branch trees, conversation management, spelling queries and shortcuts.

The New Conversation​

Three ways:

  1. Tap the “New conversation” button at the top of the side bar.
  2. The quick key. Ctrl/Cmd + Shift + O。
  3. Ctrl/Cmd + K Open the command menu and select “New conversation”.

The new conversation page (home page) displays greetings (good morning / good afternoon / good evening / late night), enters the box vertically, below a set of recommendation cards (proposals, learning plans, reading code, etc.) and points the card to chat with the tip. After the first round of response is completed, the system automatically generates a title of no more than 10 words for the conversation and can be renamed at any time.

The model picker​

The model picker is located on the right side of the input box toolbar (the moving end is at the top of the conversation bar).

  • Selection of chip.: At the top of a row of administrator-defined tags (such as “Quick”, “Multimodal” and “Reflection”), the list will only show the model with that tag, and select “All” to restore the full list.
  • List of models Each model displays icons and names, and the current model is marked with a circle.
  • Images of Group Components: List the image model separately at the bottom of the list, select to enter the drawing mode (see detailed Images are generated)。

Change the session model.​

During the conversation, you can change the model at any time, and the system tells you "Model changed: next message will use a new model."The model name is marked under each AI bubble that responds, even after the model is deleted by the administrator, the history is retained.

info

When switching between manufacturers (for example, Claude switched to GPT), the manufacturer's own historical content can not be reproduced, the system will compress the tool process into a text summary to continue the conversation, the original text and images are not affected.

Input capacity​

Text and sending.​

  • Enter to send, Shift + Enter and change, Ctrl/Cmd + Enter The same goes for sending. Enter will not be mistakenly sent in the process of entering phrases such as Chinese.
  • The send button in the generator turns to "stop generating" and click immediately to discontinue this response (part of the reservation that has already been produced).
  • A message over the max character count (12,000 by default) is never rejected: an over-length paste is automatically saved and uploaded as a .txt attachment (like Codex's “Pasted text”), and submitting an over-length draft converts it the same way — the full content still reaches the model, injected in full for small text or retrieved for very large text.
  • The draft is stored separately in the conversation cache, switching to the conversation and returning, and the text not sent remains.

Appendix is uploaded​

  • Tap the reverse needle icon to select the document, or directly tap the document. Enter the Input Box. (The "Draw the file above" message appears), or put the clipboard in the Images directly attached. come in .
  • Upload shows the percentage of progress; document class attachments then go through two stages of "resolving ..." and "indicating ...", and after completion the model can read the content.
  • The permitted file types and size limits are configured by the administrator (images and ordinary files are restricted separately), the limit will be prompted directly before uploading, not white and so on.
  • The “Documents” button at the top of the bar opens the “Documents for this conversation” panel: focus on viewing, supplementing, and removing all the files for this conversation. After removing a file, subsequent responses no longer use it, but the history message that uploaded it remains.
tip

The tables, CSV, and images uploaded are automatically stored in the conversation's Python sandbox workplace, and the model can read the analysis directly with code without any intermediate action. The Python Sandbox。

Added pictures​

Picture icons are shortcuts to "select only images" and go through the same upload channel as the attachment. The uploaded images can be viewed directly by a multi-modal model and can also be used as a reference image in the drawing mode (picture/editing).

The voice input.​

  • Touch the microphone icon to start recording (the button turns red and pulses) and stop again.
  • Transcribe automatically after the end of the recording and recognize the text Added To the input box spotlight, it will not cover the already existing content, and the input box will automatically focus after transcription is completed.
  • It is necessary for the administrator to configure the voice transcript model; the browser does not support the recording when the message "current browser does not support voice input", when rejecting the microphone permission the message "microphone permission is rejected", and the failure to transcribe the message "can not transcribe this recording."

Study Switching​

Click the "Research" button on the telescope icon and light up this message after going to the Deep Research process: multiple searches, cross-validation, generating bands reference reports. The switch remains open until you turn it off manually. The button is only displayed when the administrator opens Deep Research for the model and your user group has research permissions. Deep research。

Interrupt the school.​

The Shield icon's "Examination" button (only displayed when the administrator has configured the Examination model) is opened and the second model checks the facts when each answer is completed, and a trust badge appears next to the answer:

  • “In the school...”: verification is underway.
  • Green “Examined”: No problem found.
  • Orange "Discovering N-spot problems": A report can be launched line by line, each referring to the original sentence, describing the issue, marking the level of seriousness (error / suspicion / hint) and displaying "by X reviewed".

The switch remains open; a response is re-generated along the original round review status.

Thinking switch and model parameters​

If the administrator has configured the parameter control for the current model, the corresponding switch or drag will appear in the input box toolbar, such as the "Deep Thinking" switch, "Thinking Intensity" selector. The controls vary by model, your choices will be remembered by model and retained by model after switching.

The knowledge base selector​

The book icon opens the knowledge base selection panel, hangs one or more knowledge bases to the current conversation; after hanging the button brightens and displays the number, the model can refer to the document in which the answer is referenced by the search tool. The knowledge base。

Move the terminal.​

Under the narrow screen, the attachment / image / voice / research / knowledge base / parameter controls all enter a "+" menu, the line only retains "+", model picker and enlarged circular sending keys; when the tool is activated (research is opened, the knowledge base is selected), the "+" button has a high bright circle.

Default Tool State​

The toggles above (Research, Verify, Knowledge Base, etc.) all apply only to The current conversation. If you'd like every new conversation to start with no tool capability at all (web search, Python sandbox, image generation, knowledge base retrieval) by default, open Settings → Personalized → the “Tools” section → the “Disable tools by default” switch:

  • Once turned on, every conversation you create afterward starts with all tools off — every toggle in the composer's “+” menu begins in the off position.
  • You can still turn a tool back on temporarily for any single conversation at any time via the “+” menu, without affecting this global default.
  • This only affects the starting state of new conversations — it doesn't change the current tool toggles on conversations you already have.

  • During the stream, the page automatically switches to the bottom; after scrolling up manually, the bottom is suspended, and the "hop to the latest" button appears.

The tree navigation.​

Editing questions and re-generating answers generate real tree-shaped branches rather than covering history:

  • When there are multiple parallel branches, the message appears below. < 2/3 > Switch controls (the right arrow is "the previous branch / the next branch" respectively), switch Not interrupted ongoing exports.
  • "A branch is a new conversation" can refer to the current branch path as a separate new conversation.

Conversation Overview of Floating Window​

The branch icon in the top column opens the "Conversation Overview" floating window: list all questions by serial number (up to two rows each), with the nodes of the branch with the "N Branches" corner; click on any nodes to smoothly scroll the position. The floating window can drag, adjust the size of the lower right corner, scale by 12.5% step length, and automatically close when switching conversation.

The side preview.​

For more than 5 rounds of long conversation, the "conversation map" preview bar appears on the right side of the desktop message area: each question has a scale, the scale in the current view area is high, the suspension displays the question preview, and the direct jump is clicked.

Introduction to Conversation (Conversation)​

Respond to one of the questions in the reply without disrupting the main conversation:

  1. In the message. Select a text. The “Question” button appears.
  2. After clicking, open the Inbound input box (locate the "Question for this content..."), enter the question and send it.
  3. The system created one. determined by the text. The sub conversation is carried out in the side drawer; the original text being quoted is automatically injected as the subtext, and the sub conversation can continue independently.
  4. The original text leaves a high mark, then click the mark to reopen the corresponding sub conversation; the already marked text is selected again will open the existing sub conversation directly, will not be created again.
