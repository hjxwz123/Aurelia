Documents and Storage Space

The “Documents” page is the unified management gateway for all your uploaded content: click “Documents” under the sidebar “Draw” to enter. the attachments in the conversation, the documents in the knowledge base, are all listed on this page – no need to go back to each conversation or knowledge base to look for one by one.

The page is composed​

** The top is storage. **: Displays a red alert when space and total quota are used (configured by the administrator by user group), and progress bars exceed 90%. If your group does not have a quota, “xx is used · no limit”.

** Below is the list of documents. ** Each line includes:

Supports search by file name (without distinction between big and small), filter by source (conversation/knowledge base), sort by time or size, and load lists by page.

The online preview.​

Click "See" for direct preview, no download is required: Images are displayed directly, PDF built-in reader, text/code highlighted (more than 2 MB of text is recommended to download after viewing), and other formats are available for download.

Rules of storage quota​

  • ** Only document class documents counted in quota ** PDF, Word, tables, text and so on. ** Images are not included. ** More pictures do not take up space in the conversation.
  • The same upload shared the same byte between the conversation and the retrieval index, counting only once.
  • Continuing to upload the document when the space is full will be rejected, with the message "Storage is full, please go to the file clean page first".
  • The quota size is set by the administrator by group in admin console > user group (0 represents unlimited).

Deletion of documents​

After removal and confirmation, the document will be ** Completely removed at all locations. ** The file entity on the disk, the database record, the retrieval vector are cleared together, and cannot be restored.

Removing related conversations will not report errors, but there are two things to note:

  • The attachment tag in the historical message still appears, but the preview prompt does not exist; the abbreviation of the image attachment turns to the "file has been deleted" position.
  • ** The “Memory” model. **: When the conversation continues with the question, the model can no longer read the content of the file (the image is no longer available to the visual model, the document no longer appears in the search results), only based on the answer text that has been generated in the previous rounds. knowledge base document synonym – the conversation that loads the library after deletion can not retrieve the document, the rest of the document is not affected.
