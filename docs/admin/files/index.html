List of documents

"Data > File" is a unified view of the entire station uploaded files: all users uploaded files in conversation and knowledge base, one table viewed and one cleaned. suitable for storage audits, cleaning large files, investigating violations.

Listing and Selection​

Each line displays: file name, user (mail box), source (conversation logo or knowledge base name logo), size, upload time.

Filtering and sorting:

  • ** Name of document search ** Substrate matching, not distinguishing between large and small letters;
  • ** The User ** Draw down to view only a user's file;
  • ** Source of **: all / conversation / knowledge base ;
  • ** Ordered **: By upload time or file size, two-way.

When the same conversation is uploaded, the attached record and the retrieved document record (the same physical byte) are present in the database, and the list is automatically folded into a row and will not be repeated.

Check the content.​

Each line of the "View" button previewes the file content online: images, PDFs, text directly displayed, other formats can be downloaded.

Deletion and Deletion​

  • Select "Delete" on a single line or select "Delete selected" at the back of multiple rows to confirm that the pop-up lists the file names that will be deleted.
  • Removal is ** Three layers of cleaning. ** Database records (including retrieval parts) → vectors in the vector library → file entities on the disk. Go to the same link with user-side deletion, leaving no residues.
  • Items that no longer exist when the batch is deleted automatically skip and do not result in a whole batch of failures.
Co-operation with storage quota

在user group When you configure storage (MB) for each group, the total amount of non-images uploaded by users will be limited; the file list here can help you find the most user and file.
