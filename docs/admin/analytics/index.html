Removal affects counting.

Usage logs are not just reports: after the service is restarted, the user’s quota count and time-effectiveness credits are all re-seeded from this table. Recently Recordings (still within the quota window) may underestimate the number of times used/credits used by relevant users. Cleaning the historical archive data is not affected, and it is recommended to only delete batches according to the earlier timeframe.

Error Request and Failed Load Log​

Errors in the account.​

When the upstream call fails (connection error, 401 / 403 / 429 / 5xx, etc.) The same note. Marked as an error state, tokens, fees, credits are all zero.This makes the number of failures statistical, can be filtered according to "only errors", channel quality problems can be seen at a glance.

The calibration guarantees: Errors do not affect the user. It is excluded from the quota window, the usage count visible on the user’s side and the analytical dashboard, and failure requests neither consume the user’s free number nor infuse the “called number/active user” into fluids that do not coincide with the token/payment.

Online user judgment​

Online status is displayed in the User > User list: There is a status point in front of each line of the mailbox, green for online and grey for offline; the user information pop-up also shows the precise time of "Recent Activity".

  • The User In the last 5 minutes. Applications that have been issued arbitrarily with authentication are considered online.
  • The server updates the user's most recent active time when processing requests to control the frequency of writing. Updated every minute. Therefore, online judgment has a maximum of about 1 minute of delay.
  • The judgment is based entirely on request touch, no independent heartbeat channel: the user opens the page but for a long time without any action (no message, no page) will gradually appear as offline, which is expected behavior.
Distinguish between “active users” and analytical dashboards

The "online" list of users is This moment Nearly real-time status; the "active user" in the analytical dashboard KPI is The Statistical Window. The number of de-weights generated by the model call. The former answered "who is now in", the latter answered "who is in use during this time."
