画像生成

Aivory の画像生成は、単独の「図面ページ」ではなく、チャットの深さと統合されます:同じ会話ボックスで選択した画像 モデルを描き、チャットに戻って描き、Python サンドボックスでこの画像を処理し続けることができます。このページには、図面の入力、スタイルとヒントの最適化、参照図の編集、制限とクレジット請求、個人ギャラリー、および管理者側の各ユーザー図書館が含まれています。

前 前の投稿: 画像 モデル どこから​

画像 モデル と 会話 モデル 同じセットのチャンネル/モデル システムを共有: 管理者がチャンネルで作成kind='image'モデル 項目 (設定方法を参照)チャンネルとモデル以下の2つのチャンネルが異なります。

チャンネルタイプ生成方法編集 / 変更方法
Gemini会話の生成(generateContent画像の輸出)「紫の空に変える」を次のラウンドとして同じ画像セッションに追加し、前回の図面に基づき、主体一致性が強く
OpenAI無条件の呼び出し/v1/images/generations走 /v1/images/edits原図と指示

ユーザーはこれらの違いを認識する必要がなく、プラットフォームは統一的に吸収され、複数の編集のセッション状態はサービス側で永続化し、ページを更新し、翌日に変更することができる。

2 使用方法​

方法1:描写モデル(推奨)​

入力方法1を選択する:

  • サイドバー「絵画」入口(プロジェクト エリアの下): 図面を用意したトップページを開き、最初の可用な画像 モデルを自動的に選択します。
  • モデルピッカー: 任意 セッションでモデルピッカーを開いて、下部の「画像生成」パッケージは、すべての有効な画像モデル(アイコンと名前を含む)をリストし、描画モードに入ります。

図形モードでは、メッセージを送信するということは、図形を作成するということです。生成プロセスには、独自の進捗 UI があります:最初に「ヒントを最適化している...」と、次に「画像を描いています...」を表示し、アニメーションをポジション化し、完成後、画面が直接会話に表示されます。

生成後、処理を継続したい(カット、水印、パズル、データのビジュアルスプレー) で モデルピッカー チャットをカット モデル できる:同じセッション、生成したばかりの画像のカップルPython サンドボックス見える。

方法2:会話でモデルを描く​

チャット モデル 通過できますimage_generateツールグラフィック: 普通の会話で「私に絵を描くのを手伝ってください」と直接言い、モデルが自動的にツールを呼び出します。このパスで使用する画像モデルは「設定→モデル→デフォルト画像モデル」で事前選択します(画像モデルが利用できない場合、管理者に設定を依頼します)。

シングル セッションでは、このツールの最大呼び出し数は 8 回(ディープリーサーチモードでは 4 回)です。

2つのルートが同じ量のプールを共有する

図形パターンとチャット ツールの呼び出しは、最終的に同じ生成パイプラインで行き、同じ画像 モデルで同じ日々の限界を共有し クレジット 請求書は「異なる入力の限界を回避する」はありません。

言葉の最適化とスタイル​

言葉の最適化​

管理者は、コンソールの「設定」ページを管理して「画像ヒント モデル」を設定できます:設定後、あなたのオリジナルの説明は、より具体的な描画ヒントに拡張して生成され、構成されていない場合や呼び出しが間違っている場合、マウントをジャンプし、オリジナルのテキストを直接使用し、常にグラフを作成できます。

スタイル予想​

図形モードの入力ボックスに「スタイル」選択器(縮小図グリッド)が表示されます。

  • スタイルは管理コンソールの「画像スタイル」ページで管理者が維持し、各スタイルには名前、サンプルグラフ、順序、セクションが含まれています。ユーザーに隠されたスタイルのヒント;
  • スタイル名とサンプルスクリプトグラフを見ることができますが、後部サービス側は隠されたヒントを最終的な図形指示にペットし、ヒント自体はブラウザに送信されません。
  • 「なし」を選択すると、いかなるスタイルにも適用されず、管理者がスタイルを設定していない場合に「いかなるスタイルにも適用されない」が表示されます。

グラフを再生するには、元のスタイルに沿って選択され、再選択する必要はありません。

参考図:図形と編集​

図形モードで画像の添付ファイルをアップロードすると、入力図として参加して生成されます:

  • 自分の写真をアップロード + テキスト指示 = グラフィック / スタイルの移行;
  • 前のラウンドで生成された画像に変更を継続する指示 = 複数の編集(Gemini クラス モデルは、複数の「もう少し」の間でキャラクターとシーンを一致させます)。
  • 画像の添付ファイルには最大サイズが制限されています(管理者向け、デフォルトは5 MB)、ファイルを選択するときに超限がブロックされます。

サイズと数量​

インタフェースのサポート数(nサイズ(size)のパラメータ:

  • 会話ツールの道: リクエストを直接指示に書き込み、「16:9横幅のポスターのスケッチを2枚描く」など、モデルがそれをツールパラメータに翻訳し、サイズパラメータが有効かどうかはアップストリーム 画像 モデル自体のサポートに依存します。
  • 描くモデル: 構図要件(横版 / 垂版 / 比率 / 枚数)も同じように提示詞に書かれ、提示詞最適化 モデルは最終指令にそれを保持します。

制限額とクレジット料金​

画像の単価は高く、プラットフォームはメッセージよりもそれをコントロールし、ルールは次の順序で発効します。

  1. 毎日の上限管理者は「設定」ページで全サイトの「毎日の画像上限」を設定し、毎日のメッセージ上限とは独立しています。
  2. グループごとにモデルごとに無料: 管理者は、画像モデルごとにユーザーグループごとに無料のサイクル割り当てを設定できます。
  3. クレジット返還: 無料配当が使用終了後、自動的に割引クレジットに変更され、「単一価格 × ドル当たりのクレジット数」で割引し、チャットと同じクレジット残高を共有します。
  4. 免除と禁止管理者アカウントは割引に制限されず、画像モデルがあなたのユーザーグループに開かれていない場合、モデルピッカーに表示されません。

視覚化されたデザイン:

  • グループの無料クレジットが使えなくなり、クレジットを生成すると、モデルピッカーの「画像生成」パッケージのモデル名の右側に「Nクレジット」ロゴが表示されます(単一コスト)、頭の中には数が描かれています。
  • 請求の時点は「出図才算」:生成失敗、遮断無数次不扣分。
  • グラフごとに消費量が記録され、管理者は使用量レポートで「画像」の用途をフィルタリングできます。

クレジットシステムの完全な説明ユーザーとクォーターよくある疑問よくある質問「毎日どれだけのメッセージを送信できるか、どれだけの画像を生成できるか」

生成超時間

管理者は、画像 モデルごとに個別に「オーバータイム(秒)を生成する」を設定し、その時間を超える単一リクエストで中断(0 は無制限)します。

私のギャラリー​

サイドバー「絵」から絵画のトップページに移動すると、下にスロールすると「私のギャラリー」になります。

特性説明
コンテンツあなたのすべての生成画像、図形パターンとチャットツールの呼び出しはすべて
布局滝の表面が流れ、視野に入るときに浮かび上がる。
追加ランダムなロード:最初のページリクエストをギャラリーエリアにスロールし、その後自動的に無限にスロールしてページを分割し、それぞれの図面には独立した骨格が配置されます。
ジャンプ任意のスナップグラフをジャンプして、そのスナップグラフにある会話に戻し、編集や質問を続けることができます。
空態「まだ作品がありません、上に絵を描いてください」

ギャラリーと セッションの生成図は、ブラウザ側のキャッシュ(HTTP キャッシュ + Service Worker の画像キャッシュ-first)を有効にし、繰り返しの閲覧はもはやソースに戻らない。

よくある質問チェック​

モデルピッカーに「画像生成」分類はありません

管理者はまだ作成していません。kind='image'モデル は、作成されたが有効でないユーザー グループに開かれていません。 画像 モデル は、そのグループのユーザーのセレクターに表示されない場合に表示されません。 管理者にチャンネル/モデル ページでチェックして下さい。チャンネルとモデル。

「設定 → モデル → デフォルト画像 モデル」表示は現在利用できません。

同時に、本質的にはあなたに開かれた画像モデルがないので、この時点でチャットモデルの描写ツールも無効です。

生育時間が止まって「絵を描いている」と失敗

このモデルによって生成された「生成超時間(秒)」の制限が一度生成され、超時間は中断され、アップストリームモデルが連続またはネットワークが遅いときに引き起こされるのは簡単です。 後で再試行するか、または管理者がそのモデルの超時間の値を拡大するようにしてください。 失敗を生成するには、制限やクレジットは含まれません。 安心して再試行してください。

クレジットが足りなくなったか、クレジットが足りないか

3 つのレベルの制限:全サイトの日々の画像の上限、あなたのグループのモデルの無料割引、クレジット残高。管理者は「設定」ページと「ユーザーグループ」クォータで調整し、ユーザーはコードを交換してクレジットを追加できます。ユーザーとクォーター与よくある質問。

ギャラリーの写真は空っぽです。

ギャラリーはランダムで、リクエストを開始するためにギャラリーエリアにスロールする必要があります; 単一の図面には独立した骨格が置かれ、ロードが終わるまで空入します。 長期的な空白の場合、ネットワークとログイン状態をチェックしてからページを更新してください。

管理者側:各ユーザー図書館​

管理コンソールの「ユーザ」リストで、管理者は、ユーザのデータライブラリページにユーザ行の追加メニューを介してアクセスし、そのユーザのデータをグリッドで表示する「グリッドライブラリ」ゾーン。全画像生成(図形モードとチャットツールの呼び出しの両方のパスの出力がカバーされています):

  • 任意のマイクログラフを直接図にタップ 会議の管理者がページを表示し、生成された完全なコンテキスト(ヒント、参考図、複数の編集プロセス)を見ることができます。
  • リストは「もっとアップロードする」ページをサポートします。
  • 使用量レポートの「画像」用途フィルタリングにより、各ユーザーの生産量とコストを確認できます。
