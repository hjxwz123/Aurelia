Python サンドボックス

各セッションには独自の Python サンドボックス コンテナがあります。 モデルは回答プロセス中に Python コードを自動的に作成し、実行できます:アップロードしたデータを分析し、グラフを描き、テーブルを処理し、ダウンロード可能な PDF / PPTX / DOCX / XLSX ドキュメントを生成します。 サンドボックスはチャットと並行して動作し、ファイルはセッションのライフサイクル全体で永続的に保存されます。

このページでは、ユーザーの目に見える行動について説明します; サンドボックス サービスそのものの展開は、サンドボックス展開。

モデル いつ使う サンドボックス​

特別な操作は不要で、自然言語で直接要求を提出できます。 モデルは、以下のタスクに直面したときにコード実行ツールを独自に呼び出します。

  • 数学計算、統計分析
  • アップロードした CSV / Excel / データファイルを読み、分析します。
  • グラフを描く(matplotlibなど、中間テキストをサポート)
  • ダウンロード可能なドキュメント(レポート、スライド、表、以下「ドキュメント生成」を参照)を生成します。
  • 複数のステップのデータストリーム(データ構造をまず見て、計算し、ファイルを生成します)。

典型的な単一メッセージワークフロー:

你:"分析 data.csv,生成柱状图,再制作一份包含该图表的幻灯片。"

模型在同一轮内依次执行:
  python_execute  读取 /workspace/uploads/data.csv,输出分析结果
  image_generate  渲染柱状图,产物自动存入工作区
  python_execute  用 python-pptx 把图表嵌入 slide.pptx
  幻灯片以下载卡片出现在回复末尾

ファイルをサンドボックスに入れる方法​

コードを実行するたびに、次のコンテンツが自動的にサンドボックスに保存されます。/workspace/uploads/ディレクトリ、 モデル は、経路によって直接読むことができます:

コンテンツ説明
この会話でアップロードしたすべてのドキュメント表、CSV、PDF、画像など、最大20 MB
この会話で生成されたすべての画像例えば、前回のimage_generate描いた地図
あなたが使用する権限のあるスキル資産所在地/workspace/skills/<name>/例えば、ドキュメントテンプレート。

したがって、「ファイルをアップロードし、モデルをグラフを作成し、ドキュメントにツールを挿入する」といったツールストリームは、中間操作を必要としません。

実施過程の視覚化​

ツール呼び出しの形で実行されるコードは、答えの上の思考プロセスパネルに表示されます(パネル相互作用の詳細を参照)。会話とモデル):

  • トップステータスポイント:実行中のネズミの尾草の緑色のパルスポイントで、緑色の対比に終わり、赤い警告三角に失敗します。
  • 行尾はリアルタイムにかかる数値で、走行中に毎秒跳ね、終了後に合計時間を設定する。
  • をクリックして開く。: 上部は モデル で書かれた完全な Python コード(広いフォント、スロール可能)、下部は端末スタイルの実行出力(stdout / stderr)です。

サンドボックスの長時間の実行は、空の待機として表示されません、あなたは常にそれを実行し、どのくらいの時間を実行し、どのようなコードを実行しているかを見ることができます。

製品:グラフ、文書、データ​

サンドボックスに入る/workspace/outputs/カタログの書類は、**商品(アーティファクト)**回線回線で返信する:

製品の種類展示方法
写真(PNG等)直接内部リンクが返信に表示され、新しいタグページをクリックしてオリジナルマップを見る
他のファイル(PDF / PPTX / DOCX / XLSX / CSV など)ファイル名を含むダウンロードカードが返信の最後に表示され、クリックしてダウンロードします。

製品は永久にこのメッセージに付属します:ページの更新、分野の切り替え、共有 セッション後もダウンロードできます。

ドキュメント生成(PDF / PPTX / DOCX / XLSX)​

サンドボックス では、完全なドキュメント生成ツールチェーンを事前にインストールし、「PDF レポートを作成する / PPT を作成する / Excel をエクスポートする」を直接モデルに提出できます。

フォーマット生成方法生産特性

PPTX(視覚優先)各ページをHTMLでデザインし、スライドを合成デザインはデザインに一致し、ページはページ全体の画像であり、テキストは再編集できません。
PPTX(編集可能)プログラミング編集またはテンプレートの記入編集可能なテキスト;管理者が提供するテンプレートのスキルを向上させる

  • 視覚能力をサポートするモデル 生成後自検ページの概要(テキストが流れ、重ね、切断)、問題が自動的に修正され、品質が明らかに改善され、視覚的なモデルがサポートされていない。
  • サンドボックス 内蔵の CJK フォントで、グラフとドキュメントは中国語で正常に表示されます。
  • 管理者は、「ドキュメント テンプレート + ワークフロー 説明」をスキルにパッケージ化し、 モデル ドキュメントを生成するときに自動的にロードし、ツール コール ラインに表示されます。use_skillステップです。

セッションワークゾーン: 継続回路​

サンドボックスは、ある状態会話単位で計算する環境:

  • /workspace下記のファイルは、複数の実行の間、複数のメッセージの間で保持されます:前回に書かれた中間データ、インストールされた第三者パッケージ、その後のメッセージが直接使用できます。
  • セッション中間でモデルを切り替え、ドキュメントは引き続きサンドボックスはプラットフォーム管理であり、特定のメーカーとは関係ありません。
  • コードの実行はそれぞれ独立したプロセスです。変数不履行保留ステップごとにデータを送信するには モデル がファイルとして落格するようにしてください(モデル 自身がそれを知っています)。
  • サンドボックス コンテナが一時的にリサイクルされ、作業スペースが自動的にアーカイブされ、次回実行時に自動的に復元されます。あなたに完全に透明。
ヒント

サンドボックスチェックアウト(サンドボックスチェックアウト)/admin/sandbox) 任意の セッション ワークスペース ファイルを閲覧し、クリーニングし、ディスク占有の問題をチェックします。

ネット訪問​

サンドボックス容器公共ネットワークとの隔離(ネットワークなし)、これはセキュリティベースラインです: モデル コードはデータを外部に送信できず、内部ネットワークにアクセスできません。

  • ネットワークにアクセスする必要があるコンテンツは、プラットフォームの他のツールによって代用されます:web_fetchウェブサイトを取得し、fetch_image公共の写真をダウンロード/workspace/uploads/コードの使用について;
  • できるかpip install第三者パッケージは、デプロイターが内部の PyPI グラフィック ホワイトリストを構成しているか否かによって異なります; サンドボックス グラフィックは pandas、numpy、scipy、scikit-learn、matplotlib、openpyxl などの一般的なデータ サイエンス スタックをプレインストールし、ほとんどのタスク オープン ボックスが使用されます。

失敗と自動再試行​

  • コンテナがリサイクルされた後: セッションが失敗した場合、自動的にサンドボックスを再構築し、すべてのファイルを再保存し、透明に再試行し、エラーが表示されません。
  • **コード自体が間違っている。**例外と stderr は モデル に返され、 モデル は 通常、コードを自ら変更して再試行します(単一ラウンドの予算の範囲内)、あなたは思考プロセスパネルで毎回見ることができます。
  • 執行時間この実行は強制的に殺され、モデルに報告され、モデルは小さなタスクを解除して再試行することができます。

サンドボックスが設定されていない場合​

  • 管理者がサンドボックス サービス アドレスを設定していない場合、コード実行ツールが安全モデルデモと開発デバッグに適した単純な算術のみを行うことができます; 完全な能力を必要とします サンドボックス sidecar, 参照サンドボックス展開。
  • サーバーサイド サンドボックス との互換性、AI 応答における Pythonコードブロック右上に「起動」ボタンがあり、ブラウザ内の Pyodide エンジンによって実行されます:即時起動、ゼロバックエンド依存、出力ストリーム表示、matplotlib グラフの内蔵表示で「停止」と超時間保護をサポートします。 最初の起動には CDN からロードエンジン(約 12 MB)が必要で、その後常駐します。 このパスは既定でネットワークアクセスを禁止し、高速認証モデルで与えられたコードの一部に適しています。
