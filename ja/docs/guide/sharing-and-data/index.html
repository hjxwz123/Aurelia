共有とデータ

このページでは、Aivory の「あなたのデータ」に関連するすべての機能について説明します: セッションを作成して公開リンクを共有し、個人データをエクスポートし、他のプラットフォームやサイトからセッション履歴をインポートし、プライバシー設定ページ、およびセッション間メモリの有効化と管理。

会話 共有​

共有リンクとは何ですか?​

共有リンクは1つです。読書のみ、公開、ログイン不要会話の写真:

  • 共有内容は現在アクティブな道上のメッセージ(すなわち、チャットウィンドウで現在見ている行)には、他の支店が含まれていません。
  • 作成した瞬間の設定:**その後のニュースは、共有に表示されません。**新しいコンテンツを表示し、再度共有を実行するには、新しいスナップを古いスナップに置き換え、同時に生成することができます。新しいリンク旧リンクは無効です。
  • ショートカットは、ユーザーとアシスタントのメッセージの正文、引用、付属情報のみを保持し、アカウントのアイデンティティを含みません。
警告

リンクを共有することは公開です:リンクを取得した人は誰でも閲覧できますが、ログインする必要はありません。 機密情報を含む会話を共有しないでください。

共有リンクの作成​

2つの入り口:

入口運用経路行動
会話 ページトップ3 メニュー→「共有」「シェア セッション」ボタンを開き、「シェア リンクを作成」をタップすると、リンク、コピーボタン、「公開、ログインなしで閲覧できます」メッセージが表示され、「共有をキャンセル」ボタンが表示されます。
サイドバーの歴史リスト会話の3点メニュー→「共有」ステップ1:共有(または更新)を作成し、リンクをクリップボードに直接コピーし、「共有リンクがコピーされている」メッセージが表示されます。

モバイル端では、会話ページのトップバーのアクションシート(action sheet)にも「共有」項目があります。

訪問者が見た共有ページ​

ページのアドレスは、https://你的域名/share/<分享ID>開封後:

  • 使用とチャットページ完全に一致Markdown、コードブロック、引用ソースのレンダリングは、すべての操作ボタンを隠します(送信、再試行、編集など)。
  • ページの上部には、「共有された会話」の眉毛と「会話」のタイトルが表示され、メッセージは「ユーザー/アシスタント」の役割に分類されます。
  • 足元には「Aivory で共有」のウォーターマークが表示され、未ログインの訪問者は「Aivory 体験」の入口が表示されます。
  • 登録済みユーザー「私の会話にクローン」ボタンが表示されます:このショートフォトを自分のアカウントの新しい会話にコピーし、それに基づいて質問を続けることができます。

添付と画像生成(資産共有)​

シェアスクリプトのメッセージに含まれる資産は、匿名の訪問者も正常に閲覧できます:

資産の種類共有ページでのプレゼン
ユーザーがアップロードした画像の添付内連は写真で表示
ユーザーがアップロードした他の文書ドキュメントカードとして表示(ドキュメント名)
作成した画像(たとえば、image_generate製品)内連は写真で表示
AIが製造した他の製品ダウンロードリンクを表示

これらのドキュメントへの訪問者は、独占的な公開ルーティングを共有し、**この写真に現れた文書にしかアクセスできません。**あなたの他のアップロードコンテンツのリンクを共有することはできません。

ヒント

デプロイが古いバージョンからアップグレードされた場合: 古い共有のスクリプトに添付ファイルフィールドがアップロードされていません。再度共有アップロードされたファイルを共有ページに表示するには、AI が画像を生成するには、再共有する必要はありません。

取り消し共有​

「会話の共有」ボタンで「共有をキャンセル」をタップします。

  • 取り消し即時効力最初のリンクは 404 になります(共有ページには「共有できない」が表示されます)。
  • キャンセル後、いつでも再度共有を作成できます。

安全性​

  • 共有リンクの ID は、高ランダムトークン(192 ビット)で、暴力的に推測することはできませんし、リンクを取得しないとコンテンツが表示されません。
  • 共有ページは、共有者のアカウントのアイデンティティを漏らさない。
  • ショートカットのファイルへのアクセスは、不正利用を防ぐために独立した速度制限があります。

個人データの輸出​

ルート:メニュー → 設定 →プライバシーとデータ「すべてのデータをエクスポートする」

「エクスポート」をクリックすると、ブラウザが JSON ファイルをダウンロードします。aivory-export-2026-07-11.jsonを含む:

フィールドコンテンツ
conversationsすべてのセッション、それぞれのセッションにはタイトル、メタデータ、メッセージの完全なリストが含まれています。
memoriesあなたのすべてのメモリ
exported_at輸出時間(ISO 8601)

あなたのブラウザでローカルに組み立てられ、直接ダウンロードし、コンテンツは、現在のログインアカウント自身のデータに厳密に限定されます。

備考

このエクスポート ファイルは、このページの「インポート 会話」によって直接再認識およびインポートできます(以下のセクションを参照)、したがって、デプロイを越えて個人の会話を移動するためのメディアとしても使用できます。

インポート 会話​

ルート:設定→プライバシーとデータ→ 「インポート 会話 」を選択する.json文書です。

サポートされているフォーマット(自動識別)​

同じ入力が 2 つのフォーマットを受け入れ、解析器は手動で選択する必要がなく自動的に区別します。

フォーマット説明追加能力
他のプラットフォームからのチャットメッセージは、父子関係のマッピング構造が格納されている一般的な JSON でエクスポートされます。編集/再試用 生成された枝の木が完全に Aivory に移行します 会話の木
本駅「すべてのデータを輸出」ドキュメント前のセクションから出た{ conversations: [...], memories, exported_at }構造セッションごとに追加。model_id入力後、元のモデルで直接再生できます。

導入範囲と清掃​

導入チャットの歴史とタイトルだけを保持する残りの内容は、ブラウザ側の解析時に削除されます。

  • アップロードされたファイルと使用量レコードはインポートされません。
  • アシスタントのメインテキストの思考/状態の折りたたたみブロック、内部推理マーク、埋め込まれた base64 画像はすべて削除されます。
  • このサイトのフォーマットは、テキストのブロックをテキストにまとめ、思考、ツール呼び出し、引用、画像のブロックを同じカリキュラムで捨てます。

木の移転​

ソース ドキュメントの編集分野(最初の質問の編集に生じる複数のルートを含む)は移行する:輸入が完了すると、分岐点が表示されます。< n/m >コントロールを切り替えると、アクティブなブランドは元のファイルがその時停止した場所に並べ替わり、アクティブなフラッシュが無効になると、自動的に最後のメッセージに戻ります。

制限と結果​

制限項目上限
毎回入力するセッション数1000
各セッションのニュース数10000
単条ニュース 正文200 KB
  • 導入呼び出しなし モデル 、消費なし クレジット および割引純粋なデータを書く。
  • セッションのセクションに基づくエラー容認:セッションの解析失敗は、セッションの残りのセクションに影響を及ぼさず、完了後に「N セッションがインポートされた」というメッセージが付属し、失敗した場合は、失敗した数が付属します。
  • ソースファイルで中断された空のアシスタントが「停止」状態で返信し、インターフェイスでエラーを間違えることはありません。

プライバシー設定ページ​

ルート:設定→プライバシーとデータページは2つのブロックに分かれています:

ブロックコンテンツ
データ保存説明文: セッション あなたが接続しているサーバーに保存されます(デプロイのシナリオ、すなわち、あなた自身のサーバー); データを削除するには、管理者に連絡できます。
輸出と除去「セッションをインポートする」「すべてのデータをエクスポートする」「すべてのセッションを削除する」三行操作
危険

「すべてのセッションを削除する」永久削除すべてのセッションをクリックすると、確認ボタンで再確認し、削除後は復元できません。 個々のセッションだけを削除したい場合は、サイドバーでセッションを別々に操作してください。

メモリのスイッチと管理は、このページではなく、「設定→カスタマイズ」の下のセクションを参照してください。

メモリ​

記憶って何?​

アイドルはできます。交わりを通してあなたの長期的な事実(好み、アイデンティティ、制約など)を覚えてください。

  • 各ラウンド 会話 終了後、バックグラウンドは タスクモデル異步候補者のメモリを抽出し、あなたが答えを待つ時間を占めません。
  • 記憶は状態感覚新しい事実が書かれたとき、古い記憶が期限切れかどうかを自動的に判断する(例えば「東京に引っ越した」は「北京に住む」を期限切れにする)、期限切れの記憶は歴史的参考にすぎず、もはや現在の事実とみなされなくなる。

メモリ内容の意味:

状態インターフェイスラベル意味
ACTIVE効率現在の事実、回答時に直接適用
STALE期限歴史に基づいてではなく、現在に基づいて
UNKNOWN_CURRENT未知現時点ではまだ不確実である。
HISTORICAL_ONLY歴史のみ「以前の私」のカテゴリの質問のみ
QUERY_DEPENDENT結合問題の判断適用は、現在の質問に依存します。

オープンと閉鎖​

ルート:設定→個性化「記憶」のブロック。

  • 「メモリを有効にする」スイッチがメモリを制御する使用と保存: 終了後、すべての 会話 既存の記憶を注入したり、新しい記憶を抽出したりすることはありません。
  • このオプションはサービス端に保存され、デバイス間で有効になります。
  • 管理者は、プラットフォームレベルでメモリ機能をグローバルにオフにすることができますが、グローバルにオフにすると、全体の「メモリ」ブロックが設定に表示されません。

メモリ管理​

「メモリを有効にする」を有効にすると、同じブロック内でメモリマネージャが表示されます。

操作説明
閲覧すべてのメモリ、各ステータスバッグとオプション属性キーをリストします
フィルター「すべて / 有効 / 終了」をフィルタ
メモリを追加手動で追加:メモリの内容を記入し、属性キーと属性値を選択
編集者メモリのコンテンツまたは状態を変更する(有効に表示/有効に表示)
忘れるこのメモリを削除するには、確認が必要です。

会話中の記憶​

  • 会話中に直接「覚えておいてください」と言えば、モデルが通過します。save_memoryツールはメモリに明示的に書き込み、チャットインターフェイスはメモリが更新されていることを通知します。
  • 答えるとき、有効なメモリはシステムのヒントの一部として注入され、モデルは「あなたを知る」ことになりますが、追加のモデル呼び出しは必要ありません。

プライバシーの境界線​

  • ワークスペース内のシェア 会話 完全にメモリを使用しないあなたの個人的な記憶を注入したり、共有会話から抽出したりしないで、あなたの個人的な事実がスペースメンバーに漏洩されるのを防いでください。チームワークスペース。
  • メモリは「すべてのデータをエクスポートする」に含まれています。memoriesフィールド内では、いつでも輸出できます。
  • 各メモリは個別に削除できますが、アカウントを削除するとメモリクラスが削除されます。
