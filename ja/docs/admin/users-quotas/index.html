ユーザー、 クォータ と クレジット

このページには、「ユーザー」の3つのタグページが含まれています:ユーザー(/admin/users(ユーザーグループ)/admin/user-groups(コードを変える)/admin/redeem-codes)、およびそれに関連するグループモデルクォータ、クレジットシステム、および日々の制限。

ユーザー管理​

リストと検索​

  • 検索: 名前やメールボックスで曖昧な検索を行い、入力が停止してから 400 ms 後に自動送信されます(サーバー側のフィルタリングで、大規模なユーザー数では完全です)。
  • ページ分け: サービスエンドページ、ページあたり50記事。
  • オンライン状態: 各行の左側の円点は緑色でオンラインを表します。判定基準:過去5分以内にオーバーバンド認証を要求した(アクティブタイムは毎分に1回まで更新されます)。オフラインユーザーは「最後にオンライン」を表示し、ログインしたことがないユーザーは「ログインしたことがない」を表示します。
  • 行内徽章役割(一般ユーザー/管理者)、非デフォルト ユーザー グループ名、非active状態(ブロックなど)と、自分のアカウントをマークする「あなた」
  • リストは、引っ張り調整表示順序をサポートします。

新しいアカウント作成​

「新規ユーザー」をクリックし、以下を記入します。

フィールド要求
郵便箱必須で、有効なメールボックス形式が必要です。
名称選ぶ
コード少なくとも8文字
役割一般ユーザー / 管理者

管理者 直接作成郵便局の確認なし内部展開または同僚のための開通に適しています。

各行に鉛筆のアイコンが編集ボールウィンドウを開きます:

封鎖、解除、削除​

  • 封鎖リアルタイムでグローバルに有効:古いトークンは即座に無効、進行中の生成は中断、refresh tokenは廃止、メカニズムの詳細バックアップ総合同じ場所にあるボタンを解除します。
  • 削除ユーザーは確認ボトルウィンドウがあり、取り消すことはできません。異常な: アカウントがブロックされた後、すぐにログアウト(ブロックと同様のメカニズム)、リストに「削除中」ロゴが表示され、再生(会話、メッセージ、ファイル、ベータ、ディスクのクリーニング)はバックグラウンドのタスク分割で完了し、完了した後、行が自動的に消失する - 大きなアカウントの削除はページに閉じ込められなくなります。削除中のアカウントは「復活」されません。プロセスが再起動されると、未完成の削除は自動的に継続されます。
  • 自己保護:自分を封鎖したり削除したりすることはできません。

読み取りのみ​

各行の「もっと」メニューには、2 つの読み込みのみのブリッジ パスが含まれています。

  • 会話記録:/admin/users/:id/conversationsユーザのすべてのセッション(アーカイブを含む)をリストし、シングルセッションで完全なメッセージタイムラインを見ることができ、ユーザー側と完全に一致したレンダリング(推理ブロック、ツールカード、引用、画像を含む)を提供しますが、編集/再生入力は提供しません。
  • プロジェクトとナレッジベース:/admin/users/:id/libraryユーザーのプロジェクト、ナレッジベースのドキュメントとグラフライブラリを生成(グラフライブラリページ別にロード)。

ユーザーグループ​

ユーザー グループ は、メンバーの階層化のベンダーです:名前、価格、機能のリストがユーザーのサブスクリプションページに表示されます。モデル編集ページグループごとに配置する(次のセクションを参照)。

グループ モデル クォータ​

在モデル編集ページ(モデル > モデル > 設定) の「許可」パーティションで、チャットと image モデルをグループごとに設定します。

  • 譲渡スイッチ: オープンは、このモデルを使用できるグループを示します。**グループが割り当てられると、割り当てられていないグループはそのモデルを表示/使用できません。**すべてのオフモデルは、すべての人に無制限に開放されます。
  • サイクル: N 日または N 時間ごとに固定ウィンドウです。
  • 上限の種類二選一。
    • 回数ウィンドウ内で何回呼び出したか。
    • 割合: ウィンドウ内の モデル 割り当ての累積的なコストの上限。
  • 上限値:0 は、このサイクルに制限がないことを示します。

無料配当額を超えた行為クレジットシステムが有効である場合(credits_per_usd0 より大きい)とユーザの残高があり、この モデルはこのユーザに変換されます実際の費用でクレジット使用を継続する; クレジットが足りない場合、またはクレジットシステムが有効でない場合、リクエストがブロックされ、「超過リクエスト」が表示されます。 管理者アカウントと「クォータ行なしのモデル」にはこの制限はありません。

無料+クレジットの組み合わせゲーム

典型的な構成:Free グループのモデルに「7 日ごとに 20 回」、そのグループの「サイクルごとにクレジット」にいくつかの時効クレジットを発行します。

クレジットシステム​

2種類のクレジット​

タイプ源泉生命サイクル
時効 クレジットユーザー グループ 「サイクルごとに クレジット」 は更新サイクルにより自動的に発行されますサイクル終了未使用 廃止
永久クレジット管理者がユーザーの編集ページに直接設定するか、または「クレジット追加リンク」の外部購入後、管理者によって充電されます。決して期限

削減順序時効クレジット、永久クレジット。

交換と展示​

  • 引き下げ金 = モデル 配当に基づいて計算されたコスト(USD)×credits_per_usd。
  • モデル ピッカー ペア 行く クレジット の 会話 モデル 表示相対倍率の標識 ×N, N = (入力価格 + 出力価格) ÷ 5,小数を保持し、ユーザーが各モデルを直観的に比較できるようにします。
  • 請求された画像 モデル をクリックすると、無料利用回数がクレジットに変換されると、セレクターは モデル名の後で「Nクレジット」を表示し、N = 単一画像の価格 ×credits_per_usd(例:1ドル0.2credits_per_usd=100、表示 20 クレジット)。
  • ユーザーは、サブスクリプションページで、有効期限のクレジット残高(有効期限を含む)と永続的なクレジット残高を見ることができます。

計算時刻:発行即時、失敗は不算​

  • 正常完成: ウィンドウ クォータの回数を入力し、全額でクレジットを引き下げます。
  • ユーザー中途停止モデルがコンテンツを生成している限り、同じように計算し、生成されたトークンの実際のコストに基づいてクレジットを引き下げる。
  • 送信前に失敗またはブロック(アップストリーム エラー レポート、 クォータ 過剰、 モデレーション ブロック): 無数、無割引。

クレジット 事前チェック​

「システム > グローバル設定」の「予測および不十分なバランスを遮断するリクエスト」(デフォルトで開きます):行ってクレジットラウンドは、実際のリクエストのセットアップが完了した後、モデルに送信される前に、入力トークンを予想(固定2k出力予約を加え)してクレジットに交換し、ユーザの利用可能な残高(時効+永久)を超えると、直接「残高不足」で拒否し、アップストリーム消費を生み出さない。クレジットシステムが停止したとき、この論理は自動的に飛び跳ねます。

代わりにコード​

交換コード バルト配布メンバー:各コードに1つを割り当てるユーザーグループ一段と効果的な時間ユーザーはサブスクリプションページでコードと引き換えに変換を完了する。

生成後すぐにこのバンドのコードをすべてポップアップし、個別にコピーできます; リストページには「使用されていないすべてのコピー」と「CSVをエクスポート」(コード、グループ、状態、時間の長さ、使用された回数、バンド数、メモ、時間など)があります。

管理と状態​

  • フィルター: ステータス(すべて / 未使用 / 交換 / 無効 / 終了)とバリエーション名によりフィルタリング; ページごとに 20 件のリスト。
  • 状態未使用、一部使用(複数のコードの一部使用)、交換(使用済み)、無効、有効期限が切れた。
  • 禁止/有効化無効化は可逆的で、監査記録は保持され、交換されていないコードは即座に有効になります。
  • 削除単一のコードまたは全体のバッチを削除し、交換記録とともに削除し、取り消すことはできません。**既に認定された会員は影響を受けません。**期限は自然に戻ります。
  • 交換記録: 各コードは、為替監査(誰が、いつ、どのグループから切り替えるか)を確認できます。

交換行動と制限流​

  • 交換はユーザーに直ちに交換コードのグループがユーザーの現在のグループと異なる場合、サーバー側は確認要請を最初に返し、前端ボールウィンドウの「Xグループに切り替える」メッセージがユーザーの確認後に有効になります。
  • 終了後、自動的に変換前のユーザーグループに戻ります。
  • 防爆制限: インターフェイスを IP ストリームに制限し、デフォルトで 60 秒以内に最大 10 回、環境変数を使用できますAIVORY_API_RATE_LIMIT_REDEEM_CODE_MAX / AIVORY_API_RATE_LIMIT_REDEEM_CODE_WINDOW調整、見る進階環境変数。
代わりにコードを直接請求しない クレジット

現在のバージョンの代わりにコードが与えられています。ユーザーグループ + 時間; クレジットはユーザーグループの「サイクルごとのクレジット」を通じてグループごとに発行され、永続クレジットは管理者がユーザー編集ページで手動で調整します。 代わりにコードでクレジットを発行したい場合は、クレジットをコードに該当するユーザーグループに割り当てることができます。

日々のニュースと写真の制限​

ユーザー グループ 、 クレジット に関係なく、グローバルな1日あたりのハード リミット:

项環境変数(最初の種子開始)デフォルト値後部位置
日々のニュース上限DAILY_MESSAGE_LIMIT200システム > グローバル設定 > 日々のメッセージの上限
毎日の写真上限IMAGE_DAILY_LIMIT30システム > グローバル設定 > 日々の写真の上限

環境変数は初めてのスタート種子としてセットテーブルを書き込み、その後、バックステージを設定し、バックステージを変更し、再起動する必要はありません。環境変数リストを参照主要な環境変数。
