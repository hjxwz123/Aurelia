Python bac à sable

Chaque conversation est dotée d'un conteneur Python bac à sable indépendant. Modèle peut écrire et exécuter le code Python de manière autonome pendant la réponse: analyser les données que vous avez téléchargées, dessiner des graphiques, traiter des tables, et même générer des documents PDF / PPTX / DOCX / XLSX téléchargeables. bac à sable fonctionne en parallèle avec la conversation et conserve les fichiers tout au long du cycle de vie de la conversation.

Cette page parle du comportement visible de l'utilisateur; voir le déploiement du service bac à sable lui-même Déploiement de bac à sable。

Quand utiliser le bac à sable​

Vous n’avez pas besoin d’effectuer d’opérations spéciales, vous pouvez demander directement en langage naturel. modèle appelle automatiquement l’outil d’exécution de code lorsque vous rencontrez les tâches suivantes :

  • Calcul mathématique et analyse statistique ;
  • Lire et analyser les fichiers CSV / Excel / données que vous avez téléchargés;
  • dessiner des graphiques (matplotlib, etc.) ;
  • Créer des documents téléchargeables (rapports, diapositives, tableaux, voir ci-dessous « Créer des documents »);
  • Flux de données en plusieurs étapes (voir d'abord la structure des données, puis les calculs, puis les fichiers).

Un flux de travail typique de message unique :

你:"分析 data.csv,生成柱状图,再制作一份包含该图表的幻灯片。"

模型在同一轮内依次执行:
  python_execute  读取 /workspace/uploads/data.csv,输出分析结果
  image_generate  渲染柱状图,产物自动存入工作区
  python_execute  用 python-pptx 把图表嵌入 slide.pptx
  幻灯片以下载卡片出现在回复末尾

Comment accéder au bac à sable​

Avant chaque exécution du code, le système enregistre automatiquement le contenu suivant dans le bac à sable. /workspace/uploads/ Le catalogue, modèle peut être lu directement par le chemin:

ContenuExpliquer
Tous les documents que vous avez téléchargés dans cette conversationTableaux, CSV, PDF, images, etc., jusqu'à 20 Mo par personne
Toutes les images générées dans cette conversationPar exemple, la dernière étape image_generate Le tableau dessiné.
Vous avez le droit d’utiliser des actifs de compétencessitué/workspace/skills/<name>/ Comme le modèle de fichier.

Par conséquent, « télécharger des fichiers, générer des modèles et transférer des graphiques dans des documents » ne nécessite aucune opération intermédiaire.

Visualisation du processus d’exécution​

L'exécution du code sous la forme d'une ligne d'appel d'outils apparaît dans le panneau de processus de réflexion ci-dessus (voir le panneau d'interactions pour plus de détails). Conversation et modèle):

  • Point d'état de la tête: en cours d'exécution, le point de pouls vert de la queue de la souris devient le triangle vert et échoue pour le triangle d'avertissement rouge;
  • La queue de ligne est le compte-temps en temps réel, le battement par seconde au cours de l'exécution, la durée totale définie après la fin;
  • Cliquez sur Démarrer. Au-dessus se trouve le code Python complet écrit par modèle (tels que les polices larges, les polices à rouler) et en dessous la sortie exécutée du style terminal (stdout / stderr).

Une longue exécution du bac à sable ne se traduit pas par une attente vide, vous pouvez toujours voir combien de temps il a été exécuté, quel code a été exécuté.

Réseau Visite​

Conteneur à sable Isolation par défaut du réseau public (Pas de réseau), c’est la ligne de base sécurisée : le code modèle ne peut pas exporter de données, ne peut pas accéder au réseau interne.

  • Le contenu qui nécessite l’accès à Internet est remplacé par d’autres outils de la plate-forme : web_fetch Capture de page, fetch_image Téléchargez les photos du réseau public. /workspace/uploads/ Utilisation du code ;
  • Peut être pip install Les paquets tiers dépendent du fait que le déploiement a configuré une blanche liste d'images miroirs PyPI en interne; les miroirs bac à sable sont préinstallés avec des stacks de données communs tels que pandas, numpy, scipy, scikit-learn, matplotlib, openpyxl, et la plupart des tâches sont ouvertes.

Échec et réessai automatique​

  • Le conteneur est recyclé Lorsque la session est détectée, le bac à sable est reconstruit automatiquement, tous les fichiers sont sauvegardés, les essais sont transparents et vous ne verrez aucune erreur.
  • Le code lui-même est erroné. Les anomalies et stderr sont renvoyées au modèle, le modèle modifie généralement le code par lui-même (dans le cadre d'un budget unique) et vous pouvez voir chaque tentative dans le panneau de processus de réflexion.
  • Exécution des heures supplémentaires L'exécution est forcée et rapportée au modèle, le modèle peut réessayer la petite tâche.

Pas de bac à sable.​

  • Lorsque l'administrateur n'a pas configuré l'adresse de service bac à sable, l'outil d'exécution de code Modèle de sécurité Il suffit de faire des arithmétiques simples, adaptées à la démonstration et au développement de débogage ; une capacité complète nécessite le déploiement de bac à sable sidecar, voir Déploiement de bac à sable。
  • Complément avec le serveur bac à sable, Python dans les réponses AI Le code En haut à droite, le bouton « Exécuter » est exécuté par le moteur Pyodide de votre navigateur : instantané, dépendant du bouton arrière zéro, affichant le flux de sortie en ligne et affichant le graphique matplotlib intégré, qui prend en charge la protection « Stop » et le dépassement du temps. La première exécution nécessite un moteur de chargement CDN (environ 12 Mo) puis une permanence. Le parcours est également désactivé par défaut pour l'accès au réseau et convient aux fragments de code donnés par le modèle de vérification rapide.
