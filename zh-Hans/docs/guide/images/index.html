图片生成

Aivory 的图片生成不是一个孤立的「画图页面」,而是与聊天深度融合:在同一个对话框里选中图片模型即可绘图,画完切回聊天模型还能继续用 Python 沙箱处理这张图。本页覆盖绘图入口、风格与提示词优化、参考图编辑、限额与积分计费、个人画廊,以及管理员侧的每用户图库。

前置:图片模型从哪来​

图片模型与对话模型共用同一套渠道 / 模型体系:管理员在某个渠道下创建 kind='image' 的模型条目即可(配置方法见渠道与模型)。两类渠道的行为不同:

渠道类型生成方式编辑 / 改图方式
Gemini对话式生成(generateContent 图像输出)把「改成紫色天空」作为下一轮追加进同一图像会话,基于上一张图修改,主体一致性强
OpenAI无状态调用 /v1/images/generations走 /v1/images/edits,传原图与指令

用户不需要感知这些差异,平台统一吸收;多轮编辑的会话状态持久化在服务端,刷新页面、隔天再改都能续上。

两种使用方式​

方式一:绘图模式(推荐)​

进入方式任选其一:

  • 侧栏「绘图」入口(项目区下方):打开带绘图预设的首页,自动选中第一个可用图片模型;
  • 模型选择器:任意对话中打开模型选择器,底部的「图片生成」分组列出全部已启用图片模型(带图标与名称),选中即进入绘图模式。

绘图模式下,发送一条消息就等于生成一张图。生成过程有专属的进度 UI:先显示「正在优化提示词…」,再显示「正在绘制图片…」,配画布占位动画,完成后图片直接出现在对话里。

生成后想继续加工(裁剪、加水印、拼图、数据可视化叠加),在模型选择器切回聊天模型即可:同一个会话,刚生成的图片对 Python 沙箱可见。

方式二:聊天中让模型画图​

聊天模型可以通过 image_generate 工具画图:直接在普通对话里说「帮我画一张…」,模型判断需要画图时自动调用工具。此路径使用哪一个图片模型,由你在「设置 → 模型 → 默认图片模型」里预选(没有可用图片模型时该处会提示联系管理员配置)。

单轮对话中该工具的调用上限为 8 次(深度研究模式下为 4 次)。

两条路径共用同一额度池

绘图模式和聊天工具调用最终走的是同一个生成管线,对同一个图片模型共用同一份每日限额与积分计费,不存在「换个入口绕过限额」。

提示词优化与风格​

提示词优化​

管理员可以在管理后台「设置」页配置「图片提示词模型」(一个文本模型):配置后,你的原始描述会先由它扩写成一句更具体的绘图提示词再送去生成;未配置或调用出错时跳过润色、直接使用原文,保证总能出图。

风格预设​

绘图模式下输入框会出现「风格」选择器(缩略图网格):

  • 风格由管理员在管理后台「图片风格」页维护,每个风格包含名称、示例图、排序与一段对用户隐藏的风格提示词;
  • 你只能看到风格名称和示例缩略图,选中后服务端把隐藏提示词拼进最终绘图指令,提示词本身绝不下发到浏览器;
  • 选「无」则不套任何风格;管理员未配置风格时显示「暂无风格」。

重新生成某张图时会沿用原来选择的风格,不需要重选。

参考图:图生图与编辑​

在绘图模式下上传图片附件,它就会作为输入图参与生成:

  • 上传自己的照片 + 文字指令 = 图生图 / 风格迁移;
  • 对上一轮生成的图继续发修改指令 = 多轮编辑(Gemini 类模型在多次「再改一点」之间保持人物与场景一致);
  • 图片附件有大小上限(管理员可配,默认 5 MB),超限会在选择文件时即被提示拦截。

尺寸与数量​

生成接口支持数量(n)与尺寸(size)参数:

  • 聊天工具路径:直接把要求写进指令,例如「画 2 张横版 16:9 的海报草图」,模型会把它翻译成工具参数;尺寸参数是否生效取决于上游图片模型自身的支持情况。
  • 绘图模式:构图要求(横版 / 竖版 / 比例 / 张数)同样写在提示词里,提示词优化模型会把它保留进最终指令。

限额与积分计费​

图片单价高,平台对它的管控比消息更细,规则按以下顺序生效:

  1. 每日张数上限:管理员在「设置」页配置全站的「每日图片上限」,与每日消息上限相互独立。
  2. 按组按模型的免费额度:管理员可为每个图片模型对每个用户组配置周期免费额度。额度内生成免费。
  3. 积分回退:免费额度用完后自动改为扣积分,按「单张价格 × 每美元积分数」逐张折算,与聊天共享同一积分余额;积分也不足时拒绝生成并提示。
  4. 豁免与封禁:管理员账号不受额度限制;某图片模型若对你所在的用户组没有开放,模型选择器里不会出现它。

配套的可见性设计:

  • 当你所在组的免费额度用完、生成将要扣积分时,模型选择器「图片生成」分组里该模型名称右侧会显示「N 积分」徽标(单张成本),心里有数再画;
  • 计费时点是「出图才算」:生成失败、被拦截不计次不扣分;
  • 每张图的消耗计入用量记录,管理员可在用量报表中按「图片」用途筛选。

额度与积分体系的完整说明见用户与配额,常见疑问见常见问题中「每个用户每天能发多少条消息、生成多少张图」条目。

生成超时

管理员可为每个图片模型单独设置「生成超时(秒)」,单次请求超过该时长即中断(0 为不限)。遇到慢模型频繁超时,请管理员调大该值。

我的画廊​

从侧栏「绘图」进入绘图首页后,向下滚动即是「我的画廊」:

特性说明
内容你的全部生成图片,绘图模式与聊天工具调用产出的都在
布局瀑布流版面,滚动进入视口时逐张浮现
加载懒加载:滚动到画廊区域才发起第一页请求;之后自动无限滚动分页,每张图有独立骨架占位
跳转点任意缩略图跳回该图所在的对话,可以继续编辑或追问
空态显示「还没有作品,在上方画一张吧」

画廊与对话里的生成图都启用了浏览器端缓存(HTTP 缓存 + Service Worker 的图片 cache-first),重复浏览不再回源拉图;图片列表本身始终从服务端取最新。

常见问题排查​

模型选择器里没有「图片生成」分组

管理员尚未创建任何 kind='image' 的模型,或已创建但未启用、未对你所在的用户组开放。图片模型对某个组不可用时,该组用户的选择器里根本不会出现它。请管理员到「渠道 / 模型」页检查,见渠道与模型。

「设置 → 模型 → 默认图片模型」显示暂无可用

同上,本质是没有对你开放的图片模型。此时聊天模型的画图工具也无从生效。

生成长时间停在「正在绘制图片…」后失败

单次生成受该模型的「生成超时(秒)」限制,超时即中断;上游模型排队或网络慢时容易触发。稍后重试,或请管理员调大该模型的超时值。生成失败不计入限额、不扣积分,可放心重试。

提示额度已用完或积分不足

先核对三层限制:全站每日图片上限、你所在组对该模型的免费额度、积分余额。管理员可在「设置」页与用户组配额里调整,用户可通过兑换码补充积分。详见用户与配额与常见问题。

画廊里图片一直空着不加载

画廊是懒加载的,需要滚动到画廊区域才发起请求;单张图有独立的骨架占位,加载完才淡入。若长期空白,检查网络与登录状态后刷新页面。

管理员侧:每用户图库​

管理员在管理后台「用户」列表中,通过用户行的更多菜单进入该用户的资料库页,其中「图库」区以网格展示该用户的全部生成图片(绘图模式与聊天工具调用两条路径的产出都覆盖):

  • 点任意缩略图直达该图所在对话的管理员查看页,可以看到完整的生成上下文(提示词、参考图、多轮编辑过程);
  • 列表支持「加载更多」分页;
  • 配合用量报表的「图片」用途筛选,可以核对每用户的生成量与费用。

管理后台的整体导览见管理后台概览。
