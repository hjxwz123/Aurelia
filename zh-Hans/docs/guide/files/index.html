文件与存储空间

「文件」页是你所有上传内容的统一管理入口:侧栏「绘图」下方点击「文件」进入。对话里发的附件、知识库里的文档,都在这一页列出——不用再回到各个对话或知识库里逐个找。

页面构成​

顶部是存储用量条:显示已用空间与总配额(由管理员按用户组配置),用量超过 90% 时进度条变红提醒。若你所在的组未设配额,则显示「已用 xx · 无限制」。

下方是文件列表,每行包括:

列说明
文件名带类型图标,悬停显示完整名称
类型扩展名(PDF / DOCX / XLSX / MD / PNG…)
来源「对话」徽标,或知识库名称徽标
大小文件字节数
上传时间本地化时间
操作查看(在线预览)/ 删除

支持按文件名搜索(不区分大小写)、按来源筛选(对话 / 知识库)、按时间或大小排序,列表分页加载。

在线预览​

点击「查看」直接预览,无需下载:图片直接显示,PDF 内嵌阅读器,文本/代码高亮展示(超过 2MB 的文本建议下载后查看),其他格式提供下载。

存储配额规则​

  • 只有文档类文件计入配额:PDF、Word、表格、文本等。图片不计入——聊天里发再多图也不占空间。
  • 同一份上传在对话与检索索引间共享同一份字节,只计一次。
  • 空间满时继续上传文档会被拒绝,提示「存储空间已满,请先到文件页清理」。清理后即可继续上传。
  • 配额大小由管理员在「管理后台 > 用户组」按组设置(0 表示不限制)。

删除文件​

点击删除并确认后,文件会在所有位置被彻底移除:磁盘上的文件本体、数据库记录、检索向量一并清除,不可恢复。

删除后相关对话不会报错,但有两点值得注意:

  • 历史消息里的附件标签仍然显示,但点击预览会提示文件不存在;图片附件的缩略图会变成「文件已删除」占位。
  • 模型会"失忆":后续在该对话继续提问时,模型不再能读取这份文件的内容(图片不再提供给视觉模型、文档不再出现在检索结果里),只能依据之前轮次已生成的回答文字。知识库文档同理——删除后挂载该库的对话检索不到这份文档,其余文档不受影响。
